클로드 스킬(Skills) 기초
반복 업무를 '스킬'로 자동화하기

같은 지시를 매번 반복 입력하는 시간을 없애세요.
스킬 하나가 당신의 업무 방식을 영구적으로 바꿉니다.

🎯 난이도: 입문 ~ 중급 ⏱ 학습 시간: 60~90분 🔑 필요 플랜: Pro / Max / Team

🎯 오늘의 학습 목표

  • 클로드 스킬이 무엇이며, 왜 필요한지 개념을 이해한다.
  • 기본 제공 스킬(Pre-built Skills)을 즉시 활용할 수 있다.
  • 나만의 커스텀 스킬을 SKILL.md 파일로 직접 작성하고 등록할 수 있다.
  • 반복되는 업무 유형을 스킬로 정의하는 실전 패턴을 습득한다.
📖 개념 이해

스킬(Skills)이란 무엇인가?

클로드와 대화할 때, 여러분은 매번 비슷한 지시를 반복하지 않으신가요? "내 브랜드 톤은 이렇고, 이 형식으로 작성해야 하고, 반드시 이 항목을 포함해야 해…" 일반적인 AI 사용자는 매일 10~15분을 이런 컨텍스트 재설명에 낭비합니다.

클로드 스킬(Agent Skills)은 이 문제를 완전히 해결합니다. 스킬은 "특정 업무를 어떻게 수행할지"를 가르쳐놓은 전문 지식 패키지입니다. 한 번 만들어두면 클로드는 관련 작업이 들어올 때 자동으로 그 스킬을 인식하고 적용합니다. 슬래시 명령어(예: /briefing)로 직접 호출할 수도 있습니다.

💡 스킬의 핵심 원리: Progressive Disclosure(점진적 공개) 클로드는 스킬의 메타데이터(이름·설명)만 먼저 ~100토큰으로 스캔합니다. 관련 스킬이 있다고 판단하면 전체 내용(최대 5,000토큰)을 로드합니다. 50개의 스킬이 등록되어 있어도 현재 대화에 필요한 1~2개만 활성화되어 컨텍스트 창 낭비가 없습니다.

스킬의 핵심 구조: SKILL.md 파일

───────────────── SKILL.md 기본 구조 ───────────────── --- (프론트매터 시작) name: my-skill-name
↑ 슬래시 명령어 이름 (/my-skill-name). 소문자, 하이픈만 사용
description: 이 스킬이 무엇을 하는지, 언제 사용하는지 설명
↑ 클로드가 자동 감지에 사용하는 핵심 필드. 구체적일수록 정밀하게 작동
--- (프론트매터 끝) # 스킬 제목 ## 지시사항 (Instructions) 클로드에게 이 업무를 어떻게 수행할지 단계별로 설명
↑ 프롬프트처럼 작성 가능. 코드가 없어도 됨. 마크다운 형식
## 예시 (Examples) 올바른 결과물의 예시를 제공하면 더욱 일관된 출력을 얻을 수 있음

기초: 기본 제공 스킬 이해 및 활용

⏱ 예상 시간 15분  |  별도 설정 없이 바로 사용 가능

클로드에는 이미 활성화된 기본 제공 스킬(Pre-built Skills)이 있습니다. 설정 없이 대화 중 자동으로 작동합니다.

전문 수준의 Word 문서 생성 및 편집. 목차, 제목 스타일, 표 등 자동 포함.

데이터가 담긴 Excel 스프레드시트 생성. 수식, 서식, 차트 데이터 포함 가능.

📽️

PPTX 스킬

/pptx

테마가 있는 파워포인트 프레젠테이션 완성본 생성. 다운로드 가능한 파일 형태.

📑

PDF 스킬

/pdf

PDF 분석, 텍스트 추출, 새 PDF 생성, 양식 작성 및 병합.

적용 방법 | 기본 스킬 즉시 사용법

  1. 1
    아무 설정 없이 요청하기 기본 스킬은 별도 설정이 필요 없습니다. "Word 문서로 만들어줘", "Excel로 정리해줘"처럼 자연스럽게 요청하면 스킬이 자동 감지됩니다.
  2. 2
    슬래시 명령어로 명시적 호출 더 정확한 제어를 원하면 /docx, /xlsx 처럼 슬래시 명령어로 직접 호출합니다. 입력창에 /를 입력하면 사용 가능한 스킬 목록이 나타납니다.
  3. 3
    생성된 파일 다운로드 스킬이 실행되면 응답과 함께 파일 다운로드 링크가 생성됩니다. 실제 .docx, .xlsx, .pptx 파일로 저장 가능합니다.

🎬 실습 시나리오 01

회의록을 Word 문서로 즉시 변환하기
팀 회의를 마친 후 대화 내용을 정리한 텍스트가 있습니다. 이를 전문적인 회의록 Word 파일로 즉시 변환해보세요.
  • Claude.ai 대화창을 열고, 아래의 프롬프트를 붙여넣기 합니다.
  • 회의록 텍스트를 함께 입력하거나, 프롬프트의 [회의 내용] 부분에 붙여넣습니다.
  • 응답에 포함된 파일 링크를 클릭해 .docx 파일을 다운로드합니다.
  • Word에서 열어 형식이 제대로 적용되었는지 확인합니다.

📋 복사해서 바로 쓰는 프롬프트 #1

회의록 Word 문서 자동 생성
다음 회의 내용을 전문적인 Word 회의록으로 변환해줘. --- 회의 정보 --- [날짜]: 2025년 X월 X일 [참석자]: (이름 목록 입력) [주제]: (회의 주제 입력) --- 회의 내용 --- [여기에 회의 내용 또는 메모를 붙여넣기] --- 요구사항 --- Word 파일로 만들어줘. 아래 형식을 포함해줘: 1. 제목 및 기본 정보 (날짜, 장소, 참석자) 2. 회의 목적 요약 (1~2문장) 3. 주요 논의 내용 (번호 목록) 4. 결정 사항 (굵게 표시) 5. 액션 아이템 (담당자 | 내용 | 기한 표 형식) 6. 다음 회의 일정 전문적인 비즈니스 문서 스타일로, 다운로드 가능한 .docx 파일로 제공해줘.

데이터를 Excel 표로 즉시 변환
다음 데이터를 Excel 파일로 정리해줘. [데이터 내용 붙여넣기 - 텍스트, 표, 목록 모두 가능] 요구사항: - 첫 행은 헤더(굵게, 배경색) - 합계/평균이 필요한 열에는 수식 자동 추가 - 숫자 데이터는 천 단위 구분 쉼표 표시 - 다운로드 가능한 .xlsx 파일로 제공

중급: 나만의 커스텀 스킬 만들기

⏱ 예상 시간 30분  |  Pro/Max/Team 이상 필요

이제 핵심입니다. 여러분의 반복 업무를 스킬로 정의하면, 클로드는 이후 관련 요청이 들어올 때마다 자동으로 여러분의 프로세스대로 처리합니다.

적용 방법 | Claude.ai에서 커스텀 스킬 등록

  1. 1
    SKILL.md 파일 작성 메모장, VS Code, 또는 어떤 텍스트 편집기로든 SKILL.md 파일을 만듭니다. 아래 템플릿을 참고하세요.
  2. 2
    ZIP 파일로 압축 SKILL.md 파일을 폴더에 담고 ZIP으로 압축합니다. 폴더명은 스킬 이름과 동일하게 (예: daily-briefing/). 참고 파일(템플릿, 예시)이 있다면 함께 포함할 수 있습니다.
  3. 3
    Claude.ai 설정에서 업로드 Claude.ai 접속 → 우측 상단 프로필 → Settings(설정) → Features(기능) → Skills → ZIP 파일 업로드
  4. 4
    테스트 및 활용 새 대화를 시작하고 /스킬명을 입력하거나, 관련 요청을 자연스럽게 입력합니다. 스킬이 자동 감지되어 실행됩니다.
💡 Skill Creator 활용 (더 쉬운 방법) Claude Cowork를 사용 중이라면 별도 코딩 없이 스킬을 만들 수 있습니다. Cowork에서 "Use the skill-creator to help me build a skill for [업무 설명]"을 입력하면, 클로드가 인터뷰 형식으로 질문하며 SKILL.md를 자동 생성해줍니다.

SKILL.md 기본 템플릿

--- name: skill-name # 소문자, 하이픈 사용. 예: daily-report, email-summary description: 이 스킬이 언제, 무엇을 하는지 1~3문장으로 구체적으로 설명. 클로드가 자동 감지에 이 필드만 사용하므로 핵심 키워드 포함 필수. --- # 스킬 이름 ## 목적 이 스킬의 목적을 한 문장으로 설명 ## 처리 단계 1. [첫 번째 단계] 2. [두 번째 단계] 3. [세 번째 단계] ## 출력 형식 결과물이 어떤 형태여야 하는지 명시 예: 항상 이 구조로 응답할 것: - 제목 (굵게) - 요약 (3줄 이내) - 세부 내용 (번호 목록) - 다음 단계 (액션 아이템) ## 톤 및 스타일 전문적, 간결, 한국어, 수식 없음 등 ## 예시 출력 좋은 결과물의 예시를 여기에 제공하면 일관성이 높아짐

이메일 분류 & 요약 스킬 만들기
받은 이메일을 붙여넣으면 자동으로 중요도 분류, 핵심 요약, 회신 초안까지 생성해주는 나만의 이메일 비서 스킬을 만들어봅니다.
  • 아래 SKILL.md 내용을 복사해서 텍스트 파일로 저장 (파일명: SKILL.md)
  • email-assistant 폴더를 만들고 그 안에 SKILL.md를 넣은 뒤 ZIP 압축
  • Claude.ai Settings → Features → Skills에서 ZIP 업로드
  • 새 대화에서 /email-assistant 입력 후 이메일 내용을 붙여넣어 테스트

이메일 비서 스킬 (email-assistant/SKILL.md)
--- name: email-assistant description: 수신된 이메일을 분석하여 중요도 분류, 핵심 요약, 요청 사항 추출, 회신 초안을 제공하는 이메일 처리 스킬. "이메일 정리", "메일 요약", "답장 초안", "메일 처리" 등의 요청 시 자동 활성화. --- # 이메일 비서 스킬 ## 목적 수신된 이메일을 신속하게 분석하고, 핵심을 파악하여 효율적인 의사결정과 회신을 지원한다. ## 처리 단계 1. **중요도 분류** 먼저 이메일을 아래 기준으로 분류 - 🔴 긴급: 24시간 이내 응답 필요 - 🟡 보통: 3일 이내 처리 - 🟢 낮음: 일주일 이내 또는 참고용 2. **핵심 요약** 3줄 이내로 이메일의 핵심 내용 요약 3. **요청 사항 추출** 발신자가 나에게 원하는 것이 무엇인지 명확히 파악 4. **필요 정보 파악** 회신 전 내가 준비해야 할 정보나 확인 사항 5. **회신 초안 작성** 전문적이고 간결한 한국어 회신 초안 (길이: 이메일의 절반 이하) ## 출력 형식 반드시 아래 구조로 응답: **[중요도]** 🔴/🟡/🟢 (이유 한 줄) **핵심 요약** (3줄 이내) **발신자 요청** - (요청 1) - (요청 2) **준비할 내용** - (확인 사항) **회신 초안** --- (회신 내용) --- ## 톤 - 전문적이고 간결하게 - 한국 비즈니스 이메일 관례 준수 - 불필요한 수식어 없이 핵심 전달

심화: 강력한 스킬 설계 원칙과 팀 활용

⏱ 예상 시간 30분  |  Team/Enterprise 권장

적용 방법 | 효과적인 스킬 설계 5원칙

  1. 1
    Description에 트리거 키워드를 심어라 클로드는 description만 보고 스킬 활성화를 결정합니다. "X 요청 시", "Y 작업에 사용", "Z, W 등의 키워드가 포함된 경우"처럼 명확한 활성화 조건을 포함하세요.
  2. 2
    출력 형식을 고정하라 스킬의 가장 큰 가치는 일관성입니다. "항상 이 형식으로", "반드시 이 구조를 따를 것"처럼 출력 형식을 명시하면 팀 전체가 동일한 품질의 결과물을 얻습니다.
  3. 3
    예시(Examples)를 반드시 포함하라 좋은 출력물의 예시 1개가 수백 단어의 설명보다 효과적입니다. Before/After 예시를 포함하면 스킬 품질이 크게 향상됩니다.
  4. 4
    500줄 이내로 유지하라 스킬이 너무 길어지면 처리 효율이 떨어집니다. 내용이 많다면 스킬을 분리하거나, 핵심 지시만 포함하고 세부 예시는 별도 파일로 분리(점진적 공개 패턴)하세요.
  5. 5
    스킬을 조합하여 워크플로우를 만들어라 이메일 요약 스킬 + 보고서 작성 스킬 + 발표 자료 스킬을 연결하면 하나의 완전한 업무 자동화 파이프라인이 됩니다. 스킬들은 서로 독립적으로, 혹은 조합되어 작동합니다.
⚠️ 보안 주의사항 신뢰할 수 없는 출처의 스킬은 절대 설치하지 마세요. 스킬은 실행 환경에서 코드를 실행할 수 있습니다. 본인이 직접 만들었거나 Anthropic 공식 스킬만 사용하는 것을 권장합니다.

브랜드 보이스 스킬 (brand-voice/SKILL.md)
--- name: brand-voice description: 블로그 글, SNS 포스트, 뉴스레터, 마케팅 카피 등 모든 콘텐츠 작성 요청 시 [브랜드명]의 고유한 톤과 스타일을 적용하는 콘텐츠 작성 스킬. "포스트 써줘", "블로그", "카피", "뉴스레터", "SNS 글" 요청 시 자동 활성화. --- # [브랜드명] 브랜드 보이스 스킬 ## 브랜드 정보 - 업종: [업종 입력] - 타겟: [타겟 고객 입력, 예: 30대 직장인] - 핵심 가치: [가치 1], [가치 2], [가치 3] ## 톤 & 스타일 - **말투**: [예: 친근하지만 전문적인, 격식체 없는 존댓말] - **길이**: [예: 블로그 800~1200자, SNS 150자 이내] - **피해야 할 것**: [예: 과도한 감탄사, 전문 용어, 영어 남발] - **선호 표현**: [예: "함께", "당신의", "실제로"] ## 콘텐츠 유형별 형식 ### 블로그 포스트 1. 공감 가는 도입부 (문제 제시) 2. 핵심 내용 (소제목 포함, 3~5개 포인트) 3. 실용적인 팁 또는 예시 4. 행동 유도 마무리 ### SNS 포스트 - 첫 줄에 핵심 훅 - 이모지 2~3개 (과하지 않게) - 해시태그 3~5개 ### 이메일 뉴스레터 - 제목: 호기심을 자극하는 30자 이내 - 도입: 개인적인 에피소드 또는 최신 트렌드 - 본문: 1가지 핵심 메시지에 집중 - CTA: 명확한 행동 유도 ## 예시 출력 [여기에 본인이 마음에 드는 글 1~2개를 붙여넣기] 이 예시의 스타일을 참고하여 모든 콘텐츠를 작성할 것.

적용 방법 | 스킬 조합으로 자동화 파이프라인 구축

여러 스킬을 한 번의 대화에서 순서대로 호출하여 복잡한 업무를 완전 자동화할 수 있습니다.

  1. 1
    단일 대화에서 여러 스킬 활용 하나의 대화에서 /email-assistant로 이메일을 분석하고, 그 결과를 /weekly-report에 반영하는 식으로 스킬을 연결할 수 있습니다.
  2. 2
    종합 업무 지시 프롬프트 작성 여러 스킬을 한 번에 활용하는 마스터 프롬프트를 만들면 아침 루틴 전체를 한 번에 처리할 수 있습니다.

오늘 아침 업무 루틴을 처리해줘. 다음 내용을 순서대로 진행해: ## 1단계: 이메일 처리 (/email-assistant) 아래 이메일들을 분류하고 각각 회신 초안을 작성해줘: [이메일 1 붙여넣기] [이메일 2 붙여넣기] ## 2단계: 오늘 할 일 정리 위 이메일 분석 결과와 아래 메모를 바탕으로 오늘의 우선순위 업무 목록을 만들어줘: [오늘 해야 할 업무 메모 붙여넣기] ## 3단계: 보고서 초안 (/weekly-report) 이번 주 업무 현황: [이번 주 한 일 간단히 적기] 위 내용으로 주간 보고서 초안도 만들어줘. --- 결과물은 각 단계별로 구분해서 보여줘.

--- name: seo-optimizer description: 블로그 포스트, 웹 콘텐츠, 랜딩 페이지 등의 텍스트를 SEO 관점에서 분석하고 최적화하는 스킬. 📌 18일차 핵심 정리

스킬이란?

반복 업무의 프로세스를 SKILL.md 파일로 정의한 재사용 가능한 지식 패키지. 한 번 만들면 영구 적용.

핵심 파일 구조

frontmatter(name + description)와 지시사항 본문으로 구성. 코드 없이 마크다운 텍스트만으로 제작 가능.

등록 방법 (Claude.ai)

SKILL.md → 폴더에 담기 → ZIP 압축 → Settings > Features > Skills에서 업로드

설계 핵심

description에 트리거 키워드 포함 + 출력 형식 명시 + 예시 제공 = 일관된 고품질 결과물

기본 제공 스킬

docx / xlsx / pptx / pdf — 설정 없이 자동 활성화. 실제 파일로 다운로드 가능.
