기초 개념 · What are Artifacts?

아티팩트란? — 클로드의 "만들기" 모드

클로드는 단순히 텍스트로 답하는 것을 넘어, 실제로 실행되는 결과물을 대화창 안에서 바로 생성합니다. 이것이 아티팩트입니다.

일반 응답 vs 아티팩트 — 무엇이 다른가?

❌ 일반 응답 (텍스트 코드블록)
클로드가 코드를 텍스트로 보여줌

```html
<button>클릭</button>
```

→ 직접 복사해서 에디터에 붙여넣고 브라우저에서 실행해야 함
→ 결과물이 어떻게 보이는지 대화창에서 확인 불가
✅ 아티팩트 (실시간 렌더링)
클로드가 코드를 실행하고 결과를 대화창 오른쪽 패널에 즉시 표시

→ 버튼이 실제로 클릭 가능한 상태로 표시됨
→ 수정 요청 시 즉시 업데이트
→ 다운로드 또는 공유 링크 생성 가능

Section 2 · React 아티팩트

⚛️ React 아티팩트 — 인터랙티브 컴포넌트

차트, 데이터 시각화, 복잡한 상태 관리가 필요한 인터랙티브 도구에 최적입니다. Recharts, Lucide 등 라이브러리를 즉시 활용합니다.

React vs HTML — 언제 무엇을 쓸까?

HTML이 더 나은 경우
• 정적 페이지 (계산기, 폼, 랜딩)
• 간단한 CSS 애니메이션
• 빠른 프로토타입이 목표
• 다운로드 후 바로 사용할 때
React가 더 나은 경우
• 데이터 시각화 (Recharts 차트)
• 여러 화면 전환이 있는 앱
• 복잡한 상태 관리 필요
• 실시간 데이터 업데이트
🔑 React 아티팩트에서 사용 가능한 라이브러리
차트·그래프: recharts (막대, 라인, 파이, 에어리어 차트 등)
아이콘: lucide-react (수백 개 아이콘)
수학 계산: mathjs
데이터 처리: lodash, d3
3D·시각화: three.js
UI 컴포넌트: shadcn/ui (Alert, Card, Dialog 등)
🎭 실습 시나리오 2
상황: 월별 매출 데이터를 팀 보고에서 보여줘야 합니다. 엑셀 차트보다 보기 좋고 인터랙티브한 것이 필요합니다.

# 클로드가 최적의 아티팩트 타입을 스스로 선택하는 프롬프트 내가 원하는 결과물을 만들어줘. 가장 적합한 아티팩트 타입을 스스로 선택해서 만들어줘. [내가 원하는 것] [여기에 원하는 것을 자유롭게 설명하세요. 예: "우리 팀의 월별 독서량 현황을 보기 좋게 보여주는 것", "회사 소개 랜딩 페이지", "데이터베이스 설계도"] 선택 기준: - 인터랙티브하고 데이터 시각화 필요 → React (Recharts) - 클릭·입력 등 기본 기능 필요 → HTML - 구조화된 문서 → Markdown - 다이어그램·플로우 → SVG 또는 Mermaid - 실행할 코드 → 해당 언어 코드 아티팩트 아티팩트 만들기 전에 "어떤 타입을 선택했고 왜인지" 한 줄 말해주고 시작해줘.
