📖 핵심 개념

왜 팀이 필요한가 — 단일 AI의 한계

아무리 뛰어난 AI도 혼자 일하면 한계가 있습니다. 컨텍스트 창이 넘치거나, 한 가지 관점에서만 분석하거나, 여러 작업을 순차적으로만 처리하게 됩니다. 에이전트 팀(Agent Teams)은 이 한계를 돌파합니다.

마치 실제 회사처럼 — 한 명의 PM(오케스트레이터)이 기획하고, 전문가들(서브에이전트)이 동시에 각자의 영역을 처리합니다. 각 에이전트는 자신만의 컨텍스트 창을 가지고, 결과를 공유하며, 서로 소통합니다.

L1

병렬 워크플로우 — 여러 Claude 대화로 동시 작업

⏱ 예상 시간 30분  |  Claude.ai 웹에서 즉시 가능

Claude Code 없이도 에이전트 팀의 핵심 원리를 경험할 수 있습니다. 브라우저에서 여러 개의 Claude 대화 탭을 동시에 열고, 각각 다른 역할을 부여합니다. 결과를 직접 취합하는 방식이지만, 병렬 처리의 강력함을 바로 체험할 수 있습니다.

적용 방법 | Claude.ai 다중 탭 에이전트 팀 운용
  1. 1
    오케스트레이터 탭 열기 (메인 대화) 첫 번째 탭에 오케스트레이터 역할을 부여합니다. "너는 프로젝트 매니저야. 작업을 분배하고 결과를 통합해줘"라고 설정한 뒤 전체 프로젝트 목표를 입력합니다.
  2. 2
    역할별 전문 에이전트 탭 생성 각 탭마다 System Prompt 형식으로 역할을 명확히 부여합니다. "너는 오직 보안 리뷰만 담당하는 에이전트야"처럼 범위를 좁혀야 전문성이 높아집니다.
  3. 3
    오케스트레이터에서 작업 분해 후 각 에이전트에 배포 오케스트레이터에게 "이 프로젝트를 독립적인 작업으로 분해해줘"라고 요청합니다. 나온 목록에서 각 작업을 해당 에이전트 탭에 붙여넣어 동시에 실행합니다.
  4. 4
    결과 수집 → 오케스트레이터에게 통합 요청 각 에이전트의 결과를 복사해서 오케스트레이터 탭에 붙여넣습니다. 오케스트레이터가 충돌을 해결하고 최종 결과를 통합합니다.

신제품 마케팅 캠페인을 3명의 에이전트로 동시에 기획하기
마케팅 기획, 카피라이팅, 리서치 세 에이전트를 동시에 실행합니다. 순차적으로 하면 1~2시간 걸릴 작업을 20분 만에 완성합니다.
  • 브라우저에서 Claude.ai 탭을 4개 열기 (오케스트레이터 1 + 전문가 3)
  • 탭1(오케스트레이터): 프롬프트 #1 입력해서 작업 분해 요청
  • 탭2, 3, 4에 각각 프롬프트 #2, #3, #4 입력해서 동시에 실행 시작
  • 세 탭의 결과를 탭1 오케스트레이터에게 전달해서 통합 요청

📋 복사해서 바로 쓰는 프롬프트 #1

오케스트레이터 (탭1)
메인 대화 탭 — 작업 분해 & 통합 담당
너는 이 프로젝트의 오케스트레이터(PM)야. 역할: 프로젝트를 분석하고, 독립적으로 처리 가능한 작업으로 분해하고, 각 전문 에이전트의 결과를 통합하여 최종 아웃풋을 만든다. == 프로젝트 목표 == 신제품 "MindFlow AI 플래너 앱" 출시를 위한 마케팅 캠페인 기획서 작성. 앱 특징: AI가 일정을 자동으로 최적화하고, 집중 시간을 분석해주는 생산성 앱. 타겟: 20~35세 직장인 및 1인 창업자. 채널: 인스타그램, 유튜브, 블로그. 예산: 월 300만원. == 지금 해줄 것 == 1. 이 캠페인 기획에 필요한 핵심 작업을 3가지로 분해해줘 2. 각 작업의 담당 에이전트 역할과 정확히 무엇을 만들어야 하는지 정의해줘 3. 어떤 정보를 받으면 최종 통합이 가능한지 알려줘 * 나중에 각 에이전트의 결과물을 이 대화에 붙여넣으면 최종 통합 기획서를 만들어줄 준비를 해두어.

📋 복사해서 바로 쓰는 프롬프트 #2~4

전문 에이전트 3종 (탭2~4)
탭2 — 마케팅 전략 에이전트
너는 디지털 마케팅 전략 전문가 에이전트야. 다른 모든 것은 신경쓰지 말고 오직 마케팅 전략만 담당해. == 작업 == MindFlow AI 플래너 앱 (20~35세 직장인 타겟, 월예산 300만원)을 위한 인스타그램 + 유튜브 + 블로그 채널별 마케팅 전략 수립. 산출물 (반드시 포함): - 채널별 콘텐츠 전략 및 주 발행 빈도 - 월 300만원 예산 채널별 배분표 - KPI 및 성과 측정 방법 - 첫 달 실행 타임라인 (주차별) * 카피라이팅이나 콘텐츠 제작은 다른 에이전트가 담당. 전략의 방향과 프레임워크만 제시해줘.
탭3 — 카피라이팅 에이전트
너는 퍼포먼스 카피라이터 에이전트야. 오직 카피 작성과 메시지 개발만 담당해. == 작업 == MindFlow AI 플래너 앱 (20~35세 직장인, AI 일정 최적화 + 집중 시간 분석 기능)의 마케팅 카피 패키지 작성. 산출물 (반드시 포함): - 메인 슬로건 3가지 (각각 다른 감성 방향) - 인스타그램 캡션 3개 (피드용, 릴스용, 스토리용) - 유튜브 영상 후킹 멘트 3개 (처음 15초 안에 시청 유도) - 블로그 제목 5개 (SEO 고려) - CTA 문구 5개 변형 * 전략이나 예산은 고려하지 말고 오직 설득력 있는 언어 표현에만 집중해.
탭4 — 경쟁사 리서치 에이전트
너는 시장 조사 전문 에이전트야. 오직 경쟁 분석과 인사이트 도출만 담당해. == 작업 == 국내외 AI 플래너/생산성 앱 경쟁사 분석 및 포지셔닝 인사이트 도출. (Notion AI, Todoist, 구글 캘린더 AI 기능, Things 3 등 주요 경쟁사) 산출물 (반드시 포함): - 경쟁사 3~5개의 마케팅 메시지 및 포지셔닝 분석표 - 시장에서 아직 채워지지 않은 포지셔닝 공백 2~3개 - MindFlow가 차별화할 수 있는 핵심 USP 제안 - 경쟁사 실수 사례 및 MindFlow가 피해야 할 것들 * 분석은 사실 기반으로, 추정은 [추정]으로 표시. 실행 전략이 아닌 인사이트 도출이 목표.

에이전트 팀의 진짜 힘은 에이전트를 얼마나 잘 설계하느냐에 달려 있습니다. 역할이 명확하고, 각자의 전문 영역이 겹치지 않으며, 결과물 형식이 일관될수록 통합이 쉬워집니다.

적용 방법 | 강력한 에이전트 역할 프롬프트 작성법
  1. 1
    역할 + 제약 + 산출물 형식을 명시하라 "너는 X야" + "Y는 하지 마" + "Z 형식으로 결과를 줘". 이 세 가지가 있어야 에이전트가 혼선 없이 전문적으로 작동합니다. 특히 "무엇을 하지 말아야 하는지"를 명확히 하면 다른 에이전트 영역을 침범하지 않습니다.
  2. 2
    에이전트가 생산하는 결과물 형식을 표준화하라 오케스트레이터가 통합하기 쉽도록 모든 에이전트가 같은 형식(표, 섹션 구조, 마크다운)으로 결과를 내도록 합니다. "반드시 이 형식의 JSON/표/마크다운으로 답해줘"라고 지정합니다.
  3. 3
    에이전트 간 결과물 전달 시 컨텍스트를 포함하라 에이전트 A의 결과를 에이전트 B에게 전달할 때, 단순히 복사가 아니라 "에이전트 A가 이런 결론을 냈어. 이를 바탕으로 너의 작업을 수행해줘"라고 컨텍스트를 붙여줍니다.

4에이전트 사업계획서 시스템
에이전트 1 — 시장 분석 전문가
너는 시장 분석 전문가 에이전트야. 사업계획서의 "시장 분석" 섹션만 작성해. 사업 아이템: [여기에 사업 아이디어 입력] 작성할 섹션: 1. 시장 규모 및 성장률 (TAM/SAM/SOM) 2. 타겟 고객 세그먼트 (3가지 페르소나) 3. 경쟁 환경 분석 (직접/간접 경쟁사 각 3개) 4. 시장 진입 기회 (공백 시장, 타이밍) 5. 위험 요인 및 대응 전략 재무나 마케팅 전략은 담지 말 것. 수치에 근거를 제시하고, 추정치는 [추정]으로 표시. 표와 함께 작성할 것.
에이전트 2 — 재무 계획 전문가
너는 스타트업 재무 전문가 에이전트야. 사업계획서의 "재무 계획" 섹션만 작성해. 사업 아이템: [여기에 사업 아이디어 입력] 작성할 섹션: 1. 초기 투자 비용 상세 분류 (설비/인건비/마케팅/운영) 2. 월별 매출 예측 (1~12개월) 3. 손익분기점 (BEP) 분석 4. 3년 재무 시뮬레이션 (낙관/보통/비관 시나리오) 5. 자금 조달 계획 (자기자본/외부투자/대출) 시장 분석이나 마케팅 전략은 담지 말 것. 모든 수치는 근거와 함께. 표 형식으로 작성.
에이전트 3 — 마케팅 전략가
너는 마케팅 전략가 에이전트야. 사업계획서의 "마케팅 및 영업 전략" 섹션만 작성해. 사업 아이템: [여기에 사업 아이디어 입력] 작성할 섹션: 1. 브랜드 포지셔닝 및 핵심 메시지 2. 채널별 마케팅 전략 (온라인/오프라인) 3. 고객 획득 비용(CAC) 및 생애가치(LTV) 목표 4. 1년 마케팅 로드맵 (분기별) 5. 파트너십 및 채널 전략 재무 수치나 시장 분석은 담지 말 것. 실행 가능한 구체적 전술 중심으로 작성.
에이전트 4 — 리뷰어 (검토 후 통합 지시)
너는 엄격한 사업계획서 검토 전문가 에이전트야. 제출된 세 섹션을 비판적으로 검토하고 개선 지시를 내려. == 시장 분석 섹션 == [에이전트1 결과 붙여넣기] == 재무 계획 섹션 == [에이전트2 결과 붙여넣기] == 마케팅 전략 섹션 == [에이전트3 결과 붙여넣기] 검토 관점: 1. 각 섹션에서 논리적 오류나 근거 없는 주장 2. 세 섹션 사이의 숫자/전제 불일치 3. 투자자가 지적할 가능성이 높은 약점 4. 추가로 보완해야 할 핵심 내용 출력 형식: 섹션별로 (시장/재무/마케팅) 각각: - ✅ 잘 된 부분 - ⚠️ 보완 필요 (구체적으로) - ❌ 치명적 약점 (반드시 수정) 마지막에 통합 기획서 작성 시 주의사항 3가지

Claude Code Agent Teams — 진짜 자율 에이전트 팀

⏱ 예상 시간 35분  |  Claude Code v2.1.32+ 필요

Claude Code의 실험적 기능인 Agent Teams는 진짜 자율 에이전트 팀입니다. 에이전트들이 공유 태스크 목록을 통해 작업을 나누고, 서로 직접 메시지를 보내며, 팀장의 중재 없이 협력합니다. 2026년 2월 Opus 4.6과 함께 출시된 최첨단 기능입니다.

⚠️ 실험적(Experimental) 기능입니다 Agent Teams는 현재 실험적 기능으로, 기본 비활성화 상태입니다. Claude Code v2.1.32 이상, Pro/Max/Team/Enterprise 플랜이 필요합니다. 토큰 비용이 일반 세션의 3~4배 소요됩니다. 중요한 코드는 Git으로 백업 후 사용하세요.
적용 방법 | Agent Teams 활성화 및 첫 팀 생성
  1. 1
    Agent Teams 활성화 환경 변수 설정 또는 settings.json 수정으로 활성화합니다. 아래 코드 블록을 참고하세요. Claude Code를 재시작해야 적용됩니다.
  2. 2
    프로젝트에서 Claude Code 실행 후 팀 생성 프로젝트 폴더에서 claude를 실행하고, 자연어로 팀 생성을 요청합니다. "프론트엔드, 백엔드, 테스트 에이전트 3명으로 팀을 만들어줘"처럼 원하는 역할을 설명하면 됩니다.
  3. 3
    팀원 모니터링 및 직접 소통 Shift+Down으로 각 팀원 에이전트를 전환하며 상태를 확인하고 직접 지시할 수 있습니다. tmux 환경에서는 각 팀원이 별도 패널에 표시됩니다.
  4. 4
    종료는 항상 오케스트레이터(팀장)에게 요청 개별 팀원을 직접 강제 종료하면 메모리 누수 등 문제가 생길 수 있습니다. 반드시 팀장 세션에서 "팀을 정리해줘"라고 요청하세요.
Terminal — Agent Teams 설정
# ── 방법 1: 환경 변수로 활성화 (임시) ────────────── $ CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 claude # ── 방법 2: settings.json에 영구 등록 ──────────────── # ~/.claude/settings.json 편집: { "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } } # ── 버전 확인 (v2.1.32 이상 필요) ────────────────── $ claude --version # Claude Code 2.x.x 이상이어야 합니다 # ── 프로젝트 폴더에서 실행 ────────────────────────── $ cd my-project $ CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 claude # ── 세션 시작 후 자연어로 팀 생성 ────────────────── > 이 프로젝트를 위한 에이전트 팀을 만들어줘. 프론트엔드, 백엔드, 테스터 3명으로 구성. 각 에이전트의 역할을 명확히 해줘. # ── 팀원 전환 ──────────────────────────────────── # Shift+Down: 다음 팀원으로 전환 # 각 팀원의 상태를 확인하고 직접 지시 가능

Claude Code 세션 안에서 입력
이 프로젝트를 위한 에이전트 팀을 구성해줘. == 팀 구성 요청 == 팀원 1 — 프론트엔드 에이전트 - 역할: UI 컴포넌트 개발, 스타일링, 사용자 인터페이스 - 담당 파일: src/components/, src/styles/, src/pages/ - 사용할 도구: 파일 읽기/쓰기, 린트 - 하지 말 것: API 엔드포인트 수정, 데이터베이스 작업 팀원 2 — 백엔드 에이전트 - 역할: API 개발, 비즈니스 로직, 데이터베이스 연동 - 담당 파일: src/api/, src/services/, src/models/ - 사용할 도구: 파일 읽기/쓰기, 명령 실행 - 하지 말 것: UI 파일 수정, 클라이언트 사이드 코드 팀원 3 — 테스터/리뷰어 에이전트 - 역할: 테스트 코드 작성, 코드 리뷰, 버그 탐지 - 담당 파일: src/__tests__/, src/__mocks__/ - 사용할 도구: 파일 읽기/쓰기, 테스트 실행 - 하지 말 것: 프로덕션 코드 직접 수정 (버그 보고만) 팀을 생성한 후 각 팀원이 현재 코드베이스를 파악하도록 지시해줘. 각 팀원이 준비되면 알려줘.

비개발자를 위한 에이전트 팀 활용
Claude.ai 웹 — 보고서 작성 에이전트 팀

"2025년 국내 SaaS 시장 동향 보고서"의 데이터 및 시장 현황 섹션만 담당해. 포함할 내용: - 국내 SaaS 시장 규모 및 성장률 (최근 3년) - 주요 카테고리별 시장 분류 - 주요 플레이어 및 시장 점유율 - 투자 트렌드 모든 수치는 출처 표시. 추정치는 [추정] 표시. 표와 수치 위주로 작성. ━━ 탭3: 트렌드 분석 에이전트 ━━ 너는 기술 트렌드 분석 에이전트야. 동향 보고서의 "주요 트렌드 및 기회" 섹션만 담당해. 포함할 내용: - 2025년 두드러진 SaaS 트렌드 5가지 - AI 통합이 SaaS에 미치는 영향 - 버티컬 SaaS 성장 현황 - 2026년 예상 변화 인사이트 중심으로, 스토리텔링 방식으로 작성. ━━ 탭4: 전략 제언 에이전트 ━━ 너는 경영 컨설턴트 에이전트야. 동향 보고서의 "전략적 시사점" 섹션만 담당해. 앞서 작성된 시장 현황과 트렌드를 바탕으로: - 기업이 취해야 할 전략적 액션 5가지 - 투자 우선순위 가이드라인 - 리스크 요인 및 대응 방안 - 12개월 실행 로드맵 C레벨을 위한 결단력 있는 제언으로 작성.
💡 Agent Teams를 쓸 때 vs 쓰지 않을 때 써야 할 때: 3개 이상 파일 레이어에 걸친 변경, 팀원 간 발견 사항을 공유해야 하는 리서치, 복잡한 코드 리팩토링.
쓰지 않아도 될 때: 순차적 작업, 같은 파일 수정, 간단한 버그 수정 (일반 Claude Code가 더 효율적).

📌 24일차 핵심 정리

에이전트 팀의 본질

PM(오케스트레이터) + 전문가들(서브에이전트). 각자 컨텍스트 분리 + 병렬 처리 + 결과 통합으로 단일 AI의 한계 극복.

두 가지 접근법

Claude.ai 다중 탭(모든 플랜, 수동): 역할 부여 → 동시 실행 → 오케스트레이터 통합. Claude Code Agent Teams(Pro+, 자동): 자율 협력.

좋은 에이전트 설계

역할 + 제약 + 산출물 형식 3요소. "무엇을 하지 말 것"이 명확해야 영역 침범 없이 전문성 발휘.

Agent Teams 설정

CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 환경 변수 설정. v2.1.32+ 필요. 비용 3~4배, 복잡한 작업에만 사용.

오케스트레이터의 역할

작업 분해 → 에이전트 배포 → 결과 수집 → 충돌 해결 → 통합. 개별 에이전트가 영역을 침범하지 않도록 경계 관리.
