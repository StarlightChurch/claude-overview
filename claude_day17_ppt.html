기초 개념 · 3가지 활용 방법

클로드 × PPT — 무엇을 어떻게 할 수 있나?

파워포인트 작업에서 클로드를 활용하는 방법은 3가지입니다. 상황과 목적에 따라 가장 적합한 방법을 선택하세요.

왜 PPT 작업에 클로드가 필요한가?

❌ 기존 방식의 고통
• 빈 화면에서 "무슨 내용부터 시작하지?" 막막함
• 불릿 포인트 채우는 데 1~2시간 소비
• 논리 흐름이 연결되지 않아 반복 수정
• 동일 내용 다른 청중용으로 재작업
→ 20장 보고서에 반나절 소비
✅ 클로드 활용 방식
• 주제·청중·목적만 말하면 전체 구조 제안
• 슬라이드별 핵심 메시지+불릿 자동 생성
• 스피커 노트(발표 대본)까지 동시 생성
• 같은 내용을 "임원용·팀원용·고객용"으로 변환
→ 20장 보고서 초안 30~45분 완성

방법 1: 채팅으로 내용 설계 → 직접 PPT에 입력

claude.ai에서 슬라이드 구조와 내용을 텍스트로 받아서 직접 파워포인트에 입력하는 방법입니다. 무료 플랜에서도 사용 가능하며, 내용 설계에 특화되어 있습니다.

방법 2: 아티팩트로 HTML/MARP 슬라이드 직접 생성

클로드 아티팩트 기능으로 브라우저에서 바로 볼 수 있는 HTML 슬라이드를 생성합니다. 다운로드해서 프레젠테이션에 직접 사용하거나, 구글 슬라이드에 붙여넣기도 가능합니다.

방법 3: Claude for PowerPoint 애드인 (Pro+)

PPT 파일 안에서 클로드를 직접 호출해서 슬라이드를 자동 생성하거나 수정합니다. 엑셀 애드인과 연동하면 데이터 분석부터 발표 자료까지 원스텝 자동화가 가능합니다.

방법설치플랜출력 형태최적 용도
채팅 내용 설계불필요무료 포함텍스트 구조내용 기획·초안 작성
아티팩트 HTML불필요무료 포함즉시 실행 슬라이드빠른 시각화·공유
PPT 애드인Office 스토어Pro 이상실제 .pptx 파일완성형 보고서 제작
💡 가장 효율적인 3단계 조합
① 채팅에서 구조 설계 → "이 주제로 10장 슬라이드 구조 잡아줘"
② 아티팩트로 초안 시각화 → HTML 슬라이드로 전체 흐름 확인
③ PPT 애드인으로 완성 → 실제 파일로 변환 + 디자인 적용
애드인 없이도 ①②만으로 PPT 제작 시간을 70% 이상 줄일 수 있습니다.

# 본격 작업 전에 최적 접근법을 먼저 설계하는 프롬프트 PPT 발표 자료를 만들려고 해. 전략을 먼저 세워줘. [발표 정보] 주제/제목: [예: 2026년 2분기 마케팅 전략 보고] 발표 목적: [예: 임원진 승인 획득 / 팀원 공유 / 고객 설득] 청중: [예: C-Level 임원 5명, 평균 나이 50대] 발표 시간: [예: 15분 발표 + 5분 Q&A] 슬라이드 수: [예: 15장 이내] 현재 가진 정보: [예: 데이터 있음 / 아이디어만 있음] 전략으로 알려줄 것: ① 이 상황에 맞는 최적 슬라이드 수와 구성 비율 ② 핵심 메시지 1문장 (이 발표가 끝난 후 청중이 기억해야 할 것) ③ 가장 먼저 설득해야 할 것 (청중 관심사 분석) ④ 피해야 할 실수 (이 청중에게 흔한 실수) ⑤ 사용할 방법 추천 (채팅 설계 / 아티팩트 / 애드인)
Section 1 · 슬라이드 구조 설계

📐 슬라이드 구조 설계 — 스토리라인이 전부다

내용보다 구조가 먼저입니다. 좋은 구조 위에 내용을 채우면 발표가 설득력 있게 흘러갑니다. 클로드는 청중과 목적에 맞는 스토리라인을 설계해줍니다.

# 청중과 목적에 맞는 완전한 슬라이드 구조 자동 설계 아래 발표를 위한 최적 슬라이드 구조를 설계해줘. [발표 정보] 주제: [예: 2026년 2분기 디지털 마케팅 전략 보고] 청중: [예: CEO, CFO, 마케팅 VP — 의사결정권 있음] 목적: [예: 내년 마케팅 예산 20% 증액 승인] 발표 시간: [예: 20분 (발표 15분 + Q&A 5분)] 슬라이드 수: [15장 이내] [현재 갖고 있는 내용] [예: 올해 1분기 성과 데이터, 내년 전략 방향 초안, 경쟁사 분석 자료] [설계 요청] ① 전체 슬라이드 목록 (번호, 제목, 슬라이드 유형) ② 각 슬라이드의 핵심 메시지 1문장 ③ 각 슬라이드에 들어갈 내용 요소 (텍스트/차트/이미지) ④ 예상 발표 시간 배분 (슬라이드별) ⑤ "이 구조의 설득 포인트" — 왜 이 순서가 이 청중에게 효과적인지 마지막에 "이 구조에서 가장 중요한 슬라이드 1개"를 지목해줘.
P-02복사해서 바로 쓰는 프롬프트 — 기존 내용 → 최적 구조 재편
# 이미 갖고 있는 내용을 최적 순서로 재배열 아래 내용으로 발표 자료를 만들려는데, 최적 슬라이드 순서로 재편해줘. [현재 갖고 있는 내용들 (순서 무관)] --- 아래에 갖고 있는 내용을 순서 상관없이 나열하세요 --- 예시: - Q1 매출이 전년 대비 23% 성장 - 주요 경쟁사 3사의 전략 변화 - 신제품 출시 6개월 로드맵 - 고객 만족도 조사 결과 (NPS 42점) - 마케팅 채널별 ROI 분석 - 팀 확충 계획 (2명 채용) - 예산 증액 요청 (15% 증가) - 2분기 집중 전략 3가지 [발표 맥락] 청중: [누구인지] 최종 목적: [발표 후 원하는 결과] 재편 후 제공: ① 추천 슬라이드 순서 (각 슬라이드 제목 + 들어갈 내용) ② 현재 갖고 있는 내용에서 빠진 것 (추가 필요 슬라이드) ③ 삭제하거나 부록으로 옮길 내용 ④ 전체 스토리 흐름 요약 (3줄)
Section 2 · 장표 내용 생성

✍️ 장표 내용 자동 생성 — 슬라이드별 텍스트 완성

구조가 정해졌다면 이제 각 슬라이드의 텍스트를 채울 차례입니다. 클로드는 불릿 포인트, 헤드라인, 스피커 노트를 동시에 생성해줍니다.

좋은 슬라이드 텍스트의 3원칙

❌ 슬라이드에 쓰면 안 되는 것
• 긴 문단 텍스트 (발표자가 읽는 용)
• 의미 없는 불릿 나열 ("좋은 성과를 거뒀다")
• 모든 것을 다 담으려는 욕심
• 수식어 남발 ("획기적인", "최고의")
→ 독자가 슬라이드 읽느라 발표자를 안 봄
✅ 슬라이드에 들어가야 할 것
• 핵심 메시지 헤드라인 (1문장)
• 근거가 되는 수치·데이터 불릿 (3개 이내)
• 시각적 요소 여백
• 말하지 않아도 이해되는 자명한 구조
→ 슬라이드 = 키워드, 발표자 = 설명

# 마음에 안 드는 특정 슬라이드만 집중적으로 개선 이 슬라이드 내용을 더 강력하게 개선해줘. Section 4 · 보고서 & 임원 발표

💼 보고서 & 임원 발표 장표 — 설득의 기술

임원 발표는 강의와 완전히 다릅니다. 더 짧고, 더 직접적이며, 숫자가 중심이 되어야 합니다. 클로드가 임원의 시간을 존중하는 구조를 만들어줍니다.

임원이 PPT를 보는 방식 — 알아야 할 3가지

슬라이드를 읽지 않는다 — 스캔한다
임원은 각 슬라이드에 3~5초를 씁니다. 헤드라인만 봐도 핵심을 알 수 있어야 합니다. "Q1 매출 현황" ❌ → "Q1 매출 23% 성장, 목표 초과 달성" ✅
결론을 먼저 원한다 — 두괄식 필수
과정·배경을 먼저 설명하고 결론을 나중에 말하는 방식은 임원 청중에게 역효과입니다. 첫 슬라이드에 결론을, 나머지 슬라이드에 근거를 담으세요.
무엇을 결정해줘야 하는지 명확해야 한다
발표의 목적이 "정보 공유"인지 "승인 요청"인지 "예산 결정"인지를 마지막 슬라이드에서 명확히 해야 합니다. "무엇을 결정해주셔야 합니다"를 직접 말하세요.

[발표 정보] 요청 사항: [예: 신규 AI 도구 도입 예산 5천만원 승인] 발표 대상: [예: CEO, 재무담당 부사장] 발표 시간: [예: 10분] 현재 보유 근거: [예: 파일럿 테스트 결과, 비용 절감 계산, 경쟁사 도입 현황] [10장 구조로 만들어줘] 슬라이드 1 — 제안 요약 (Executive Summary) : 요청 사항 + 핵심 근거 3가지 + 기대 효과를 딱 1장으로 슬라이드 2~3 — 문제·기회 정의 : "지금 행동하지 않으면 어떤 비용이 발생하는가" 슬라이드 4~5 — 솔루션 및 근거 : 수치·데이터 기반 설득 슬라이드 6~7 — 실행 계획 : 타임라인, 담당자, 마일스톤 슬라이드 8 — 리스크 & 대응 방안 : 임원이 물어볼 반론을 선제 차단 슬라이드 9 — ROI & 기대 효과 : 투자 대비 수익 수치화 슬라이드 10 — 요청 사항 (Ask Slide) : "이 자리에서 결정해주셔야 할 것"을 명확히 [각 슬라이드] 헤드라인(1문장) + 불릿(3개 이내) + 스피커 노트(2문장)
P-08복사해서 바로 쓰는 프롬프트 — 같은 내용 청중별 3버전 변환
# 하나의 내용을 임원/팀원/고객용으로 각각 최적화 아래 내용을 3가지 청중별 버전으로 각각 변환해줘. [원본 내용] --- 기존 슬라이드 내용 또는 핵심 데이터를 붙여넣으세요 --- [버전 A — 임원/경영진용 (5분 발표)] 포커스: 의사결정에 필요한 것만 (WHY & WHAT) 강조: 비용·수익·전략적 의미 길이: 5장 이내 어조: 결론 먼저, 직접적 [버전 B — 실무팀원용 (15분 발표)] 포커스: 어떻게 실행하는지 (HOW) 강조: 역할 분담, 프로세스, 일정 길이: 15장 어조: 협력적, 액션 중심 [버전 C — 외부 고객/파트너용 (10분 발표)] 포커스: 그들에게 어떤 가치가 있는지 (WHAT'S IN IT FOR ME) 강조: 고객 혜택, 신뢰성, 사용 사례 길이: 10장 어조: 설득적, 신뢰 구축 각 버전의 표지 제목과 핵심 헤드라인 3개도 각각 만들어줘.

🤖 Claude for PPT 애드인 — 파일 안에서 직접 생성

애드인 설치 & 시작 방법

파워포인트 → 홈 탭 → 추가 기능 → "Claude" 검색

슬라이드 생성 또는 편집 명령
텍스트를 선택하고 "이것을 슬라이드로 변환", 또는 "3~5번 슬라이드 내용을 요약해줘" 같은 명령을 직접 입력합니다.

# 엑셀에서 복사한 데이터를 PPT 장표로 직접 변환 아래 엑셀 데이터를 분석해서 임원 보고용 PPT 내용으로 바꿔줘. [엑셀 데이터 — 여기에 붙여넣기] --- 엑셀에서 Ctrl+C 하고 여기에 Ctrl+V --- [발표 맥락] 청중: [예: CEO, CFO] 발표 목적: [예: Q1 실적 보고 + Q2 전략 승인] 시간: [예: 15분] [요청 1 — 인사이트 추출] 이 데이터에서: ① 가장 중요한 인사이트 5가지 (각 1문장, 수치 포함) ② 좋은 뉴스 3가지 (임원이 기뻐할 것) ③ 나쁜 뉴스 or 위험 신호 2가지 (선제 대응 필요) ④ 즉시 행동이 필요한 것 1가지 [요청 2 — 슬라이드 변환] 위 인사이트를 활용해서 15장 PPT 내용을 만들어줘. (각 슬라이드: 제목 + 헤드라인 + 불릿 3개 + 스피커 노트 2문장) [요청 3 — 핵심 수치 슬라이드] 임팩트 있는 Big Number 슬라이드 3개도 만들어줘. (각각: 핵심 수치 1개 + 그 의미 1문장 + 맥락 1문장)
P-12복사해서 바로 쓰는 프롬프트 — 발표 대본 & Q&A 준비
# PPT 완성 후 발표 대본과 예상 Q&A 준비 완성된 PPT를 바탕으로 발표 준비를 도와줘. [슬라이드 내용] --- 완성된 슬라이드 내용 붙여넣기 (텍스트 버전) --- [요청 1 — 발표 대본] 각 슬라이드별 발표 대본을 작성해줘: - 이 슬라이드에서 말해야 할 핵심 (30~60초 분량) - 슬라이드 넘기는 자연스러운 전환 문장 - 강조 포인트 (이 슬라이드에서 한 가지를 강조한다면) 전체 대본 맨 앞에 "오프닝 멘트 (30초)"와 맨 끝에 "클로징 멘트 (30초)"도 추가해줘. [요청 2 — 예상 Q&A] 이 발표를 들은 [청중]이 할 것 같은 질문 10개와 모범 답변: - 날카로운 질문 5개 (데이터·논리 공격) - 실행 관련 질문 3개 (어떻게 할 것인가) - 리스크 관련 질문 2개 각 질문에: 답변 핵심 (2~3문장) + "절대 이렇게 답하지 말 것"
P-13복사해서 바로 쓰는 프롬프트 — 피드백 받은 PPT 빠른 수정
# 팀장/동료 피드백을 PPT에 반영하는 프롬프트 PPT에 대한 피드백을 받았어. 반영해줘. [현재 PPT 내용] --- 현재 슬라이드 내용 붙여넣기 --- [받은 피드백] --- 팀장/동료가 준 피드백을 그대로 붙여넣거나 요약 --- 예시 피드백: - "3번 슬라이드가 너무 복잡해. 두 장으로 나눠" - "ROI 수치가 없어. 추가해줘" - "결론이 약해. 더 강하게 끝내야 해" - "6번 슬라이드 순서를 7번 뒤로 옮겨" [수정 요청] ① 각 피드백을 반영한 수정 내용 ② 피드백 구현이 어렵거나 상충되는 경우 대안 제안 ③ 수정 전/후 해당 슬라이드 내용 비교 ④ 수정 후 전체 흐름이 여전히 자연스러운지 점검
P-14복사해서 바로 쓰는 프롬프트 — 마스터: 30분 안에 PPT 초안 완성
# 이 프롬프트 하나로 30분 내 PPT 초안을 완성하는 마스터 프롬프트 30분 안에 발표용 PPT 초안을 완성하고 싶어. 모든 것을 한 번에 해줘. [발표 정보] 주제: [한 문장으로] 청중: [누구인지 + 규모] 발표 시간: [X분] 목적: [정보 공유 / 설득·승인 / 교육 중 하나] 핵심 메시지: [이 발표로 전달할 가장 중요한 것 1가지] 보유 자료: [있는 것: 데이터, 사례, 아이디어 등] [한 번에 만들어줄 것] Phase 1 — 구조 (2분): □ 슬라이드 목록 (번호 + 제목 + 유형) □ 전체 스토리 흐름 요약 (3줄) Phase 2 — 내용 (20분): □ 각 슬라이드 헤드라인 + 불릿 3개 □ 각 슬라이드 스피커 노트 (2문장) Phase 3 — 완성도 (8분): □ 가장 약한 슬라이드 강화안 □ 예상 질문 3가지 & 답변 □ 오프닝 & 클로징 멘트 모든 내용은 [청중]의 언어로, 결론 먼저 쓰는 두괄식으로 작성해줘.
