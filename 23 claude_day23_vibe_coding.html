바이브 코딩이란? — "느낌대로 만드는 개발"

2025년 AI 코딩 도구의 급격한 발전으로 탄생한 새로운 개발 패러다임입니다. 바이브 코딩(Vibe Coding)은 코드의 세부 사항을 이해하지 않아도, "이런 느낌의 서비스가 필요해"라는 자연어 설명만으로 AI가 실제 작동하는 코드를 생성하는 방식입니다.

핵심은 사용자가 "무엇을(What)"을 정의하고, AI가 "어떻게(How)"를 처리한다는 것입니다. 비개발자도 아이디어를 즉시 실제 웹 서비스로 구현할 수 있습니다.

01
느낌으로 설명하라
"에어비앤비처럼 깔끔하고", "애플 웹사이트 스타일로", "미니멀하고 전문적으로" — 기술 용어보다 감각적 표현이 더 정확할 때가 많다.
02
반복으로 완성하라
첫 결과물이 완벽할 필요 없다. "헤더 색상을 더 진하게", "버튼을 오른쪽으로" 같은 작은 수정을 반복하면 원하는 결과에 도달한다.
03
구체적일수록 좋다
막연한 요청보다 "25~35세 직장인을 타겟으로 하는 커리어 코칭 랜딩 페이지"처럼 대상, 목적, 스타일을 명확히 하면 훨씬 좋은 결과가 나온다.

L1

첫 웹 페이지 만들기 — 30분 안에 완성

⏱ 예상 시간 25분  |  코딩 지식 전혀 불필요

처음으로 "내가 만든" 웹 페이지를 경험하는 순간은 생각보다 훨씬 빠르게 옵니다. Claude.ai에서 프롬프트 하나를 입력하면 1분 안에 실제 브라우저에서 보이는 웹 페이지가 완성됩니다.

적용 방법 | Claude.ai에서 웹 페이지 만드는 법
  1. 1
    Claude.ai 접속 → 새 대화 시작 claude.ai에서 로그인 후 새 채팅을 시작합니다. 별도 설정 없이 바로 웹 페이지 제작이 가능합니다.
  2. 2
    만들고 싶은 것을 구체적으로 설명한다 누구를 위한, 어떤 목적의, 어떤 느낌의 페이지인지 설명합니다. 레퍼런스 사이트를 언급하면 더욱 효과적입니다. 아래 프롬프트 예시를 그대로 복사해서 사용해도 됩니다.
  3. 3
    Artifacts 패널에서 실시간 미리보기 응답 오른쪽에 Artifacts 패널이 열리면 "Preview" 탭을 클릭해 실제 모습을 확인합니다. 마음에 안 드는 부분이 있으면 바로 대화창에 수정 요청을 입력합니다.
  4. 4
    반복 수정으로 완성도 높이기 "헤더 배경색을 진한 남색으로", "버튼에 호버 애니메이션 추가", "모바일에서도 잘 보이게" 등 자연어로 계속 수정을 요청합니다. 완성 후 HTML 파일로 다운로드합니다.
🎬 실습 시나리오 01
내 소개 페이지를 5분 만에 만들기
가장 간단하지만 가장 실용적인 시작. 자신을 소개하는 단일 페이지 웹사이트를 만들어봅니다. 포트폴리오, 명함 대용, SNS 링크 모음 등 다양하게 활용 가능합니다.
  • Claude.ai 열기 → 프롬프트 #1 복사해서 본인 정보로 수정 후 전송
  • 오른쪽 Artifacts 패널 → Preview 탭에서 결과 확인
  • 마음에 안 드는 부분 3가지를 찾아서 수정 요청 (프롬프트 #2 참고)
  • 완성 후 Artifacts 패널 우측 상단 다운로드 버튼 클릭 → HTML 파일 저장
  • 저장된 파일을 브라우저에서 직접 열어서 확인

📋 복사해서 바로 쓰는 프롬프트 #1

개인 소개 페이지
Claude.ai 대화창에 바로 붙여넣기
나를 소개하는 웹 페이지를 만들어줘. --- 내 정보 --- 이름: [이름 입력] 직업/역할: [예: 마케팅 기획자 / 프리랜서 디자이너 / 대학원생] 소개 문장: [자신을 한 문장으로 설명] 주요 스킬/관심사: [예: 브랜드 마케팅, UX 리서치, 데이터 분석] 연락처: [이메일 또는 링크] SNS: [있다면 링크드인, 인스타그램 등] --- 디자인 요구사항 --- 스타일: 미니멀하고 전문적인 느낌. 어두운 배경(다크 테마). 색상: 메인 컬러는 보라색 계열. 레이아웃: 스크롤 없이 한 화면에 모든 정보가 보이게. 애니메이션: 부드러운 페이드인 효과로 등장하게 해줘. --- 포함할 섹션 --- 1. 이름과 역할이 크게 보이는 헤더 2. 짧은 자기소개 (2-3문장) 3. 주요 스킬 태그 (배지 형태로) 4. 연락처 버튼 HTML, CSS, JavaScript를 하나의 파일로 만들어줘. 모바일에서도 잘 보이는 반응형으로.

수정 요청 패턴
결과물 받은 후 — 이렇게 수정을 요청하세요
━━━ 색상/디자인 수정 ━━━ 배경색을 더 진한 네이비(#0a0e1a)로 바꿔줘. 텍스트는 밝은 흰색으로 유지해줘. ━━━ 레이아웃 수정 ━━━ 이름을 화면 정중앙에 배치하고, 아래에 타이핑 효과로 "마케터 / 기획자 / 크리에이터"가 번갈아가며 나타나는 애니메이션 추가해줘. ━━━ 콘텐츠 추가 ━━━ 스킬 섹션 아래에 최근 프로젝트 3개를 카드 형태로 보여주는 섹션을 추가해줘. 각 카드에는 프로젝트명, 설명 1줄, 태그가 들어가게. ━━━ 인터랙션 추가 ━━━ 스킬 태그에 마우스를 올리면 해당 스킬에 대한 짧은 설명이 툴팁으로 나타나게 해줘. ━━━ 반응형 개선 ━━━ 모바일(세로 화면)에서 봤을 때 글씨가 너무 작고 버튼이 겹쳐 보여. 모바일 레이아웃을 최적화해줘.

적용 방법 | 좋은 결과물을 위한 프롬프트 작성 공식
  1. 1
    WHAT + WHO + HOW = 완벽한 프롬프트 WHAT(무엇): "카페 랜딩 페이지" / WHO(누구를 위한): "20~30대 감성 고객 타겟" / HOW(어떻게): "따뜻하고 아늑한 느낌, 갈색 톤". 이 세 가지를 명확히 하면 훨씬 좋은 결과가 나옵니다.
  2. 2
    레퍼런스 URL을 언급하라 "notion.so 같은 클린한 스타일로", "stripe.com처럼 그라데이션이 있는 다크 테마로" — 실제 웹사이트를 레퍼런스로 제시하면 클로드가 해당 스타일을 학습해 더 정확하게 구현합니다.
  3. 3
    섹션 단위로 나눠서 요청하라 한 번에 모든 것을 만들기보다, 헤더 → 서비스 소개 → 가격표 → 후기 → CTA 순으로 섹션별로 만들고 확인하면서 완성도를 높입니다. 수정하기도 훨씬 쉽습니다.
  4. 4
    "더 ~하게" 형식의 수정 요청 "더 고급스럽게", "더 젊고 역동적으로", "더 신뢰감 있게", "더 미니멀하게" — 형용사로 방향을 주면 클로드가 알아서 여러 요소를 조정합니다. 구체적인 CSS 값을 몰라도 됩니다.

적용 방법 | 인터랙티브 앱을 만드는 고급 기법
  1. 1
    로컬 스토리지(LocalStorage)로 데이터 저장 "브라우저를 닫아도 데이터가 남아있게 로컬 스토리지에 저장해줘"라고 하면 새로고침해도 데이터가 유지됩니다. 간단한 메모 앱, 설정 저장, 즐겨찾기 기능에 활용 가능합니다.
  2. 2
    공개 API를 연결해서 실시간 데이터 표시 "날씨 API(OpenWeatherMap)로 현재 날씨를 보여줘"처럼 공개 API를 연결하면 실시간 데이터를 표시할 수 있습니다. API 키 없이 쓸 수 있는 공개 API도 많습니다.
  3. 3
    Claude Code로 전환해서 완전한 웹앱 개발 단일 HTML 파일의 한계를 넘으려면 Claude Code를 사용합니다. "React로 만들어줘", "Node.js 백엔드를 추가해줘" 등 복잡한 구조도 가능합니다. 22일차 내용과 연결됩니다.
  4. 4
    무료 배포 — GitHub Pages / Netlify 완성한 HTML 파일을 인터넷에 올리는 가장 쉬운 방법은 Netlify Drop입니다. app.netlify.com/drop에 파일을 드래그만 하면 30초 만에 실제 URL이 생깁니다.

심화 | Claude Code와 연결해서 풀스택으로 확장하기
  1. 1
    Claude.ai → Claude Code로 이전하는 시점 단일 HTML 파일이 1,000줄 이상 되거나, 로그인 기능이 필요하거나, DB 연동이 필요하면 Claude Code로 전환합니다. Claude Code에서 "이 HTML 파일을 React 프로젝트로 변환해줘"라고 하면 됩니다.
  2. 2
    Claude Code에서 바이브 코딩하는 방법 Claude Code를 실행하고 "Next.js로 [서비스 설명] 앱을 만들어줘"라고 하면, 프로젝트 생성부터 코드 작성, 실행까지 자동으로 처리합니다. 복잡한 기능도 자연어로 요청하면 됩니다.
  3. 3
    Vercel로 자동 배포 파이프라인 구성 GitHub와 Vercel을 연결하면 코드를 올릴 때마다 자동으로 배포됩니다. Claude Code에서 "Vercel 배포를 위한 설정 파일 만들어줘"라고 하면 자동으로 구성해줍니다.
💡 바이브 코딩의 현실적인 한계 — 그리고 해결책 바이브 코딩으로 만든 코드는 간혹 예상치 못한 버그가 있을 수 있습니다. 해결책은 간단합니다: 그 버그를 클로드에게 보여주고 "이거 왜 이러는지 고쳐줘"라고 하면 됩니다. 진짜 코딩 실력이 없어도 "무엇이 잘못됐는지 설명"하는 능력만 있으면 됩니다. 에러 메시지를 스크린샷으로 찍어서 붙여넣는 것도 완벽한 방법입니다.

시작 도구

초보자는 Claude.ai Artifacts에서 시작. 미리보기 바로 가능. 완성 후 HTML 다운로드 → Netlify 무료 배포.

좋은 프롬프트 공식

WHAT(무엇) + WHO(누구를 위한) + HOW(어떤 느낌)의 조합. 레퍼런스 사이트 URL 언급이 가장 효과적.

수정 전략

"더 ~하게" 형식으로 방향을 주면 AI가 알아서 여러 요소를 조정. 섹션 단위로 반복 수정하면 완성도 ↑.

만들 수 있는 것

랜딩 페이지, 포트폴리오, 퀴즈 앱, 계산기, 할 일 앱, 링크 모음 페이지. 데이터 저장은 LocalStorage 활용.

내일 예고

24일차: 에이전트 팀 — 여러 서브 에이전트를 만들어 복잡한 프로젝트를 분업하는 방법.
