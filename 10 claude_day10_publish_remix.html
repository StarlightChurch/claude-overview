기초 개념 · Publish vs Remix

Publish & Remix — 무엇이고 왜 중요한가?

아티팩트를 "만드는 것"에서 "나누고 발전시키는 것"으로 — 이 두 기능이 클로드를 개인 도구에서 협업 플랫폼으로 확장시킵니다.

🚀 Publish — 내 아티팩트를 세상에 공개 🎛 Remix — 남의 아티팩트를 내 것으로 개조

Publish란?

클로드에서 만든 아티팩트를 외부 링크로 공유할 수 있는 기능입니다. 링크를 받은 사람은 Claude 계정이 없어도 해당 아티팩트를 브라우저에서 바로 볼 수 있습니다.

단순한 코드 공유가 아닙니다 — 완성된 웹 앱, 인터랙티브 대시보드, 게임이 그 자체로 URL 하나가 되어 누구와도 공유됩니다.

Remix란?

다른 사람이 Publish한 아티팩트를 보다가 마음에 드는 것을 발견했을 때, 그것을 클로드로 가져와서 수정하는 기능입니다. "이 대시보드에 우리 데이터로 바꿔줘"처럼 다른 사람의 결과물을 출발점으로 삼을 수 있습니다.

왜 중요한가 — Publish & Remix가 만드는 3가지 변화

클로드 없이도 공유 가능
팀원, 고객, 파트너가 Claude 계정 없이도 당신이 만든 결과물을 사용할 수 있습니다. 계산기, 대시보드, 게임을 URL 하나로 전달하세요.
바퀴를 다시 발명하지 않는다
이미 잘 만들어진 아티팩트를 리믹스하면 처음부터 만드는 시간 없이 더 나은 결과물을 빠르게 완성할 수 있습니다. 오픈소스처럼 서로의 작업 위에 쌓아갑니다.
팀 협업의 새로운 방식
기획자가 Publish한 프로토타입을 개발자가 Remix해서 개선하고, 디자이너가 다시 Remix해서 시각을 바꾸는 — 새로운 협업 루프가 생깁니다.

Section 1 · Publish 완전 가이드

🚀 Publish — 아티팩트를 링크로 공유하기

만든 아티팩트를 URL 하나로 세상에 공유하는 전체 과정을 단계별로 익힙니다.

1-1. Publish 버튼 찾기와 공유 링크 생성

아티팩트 패널 상단 오른쪽 → 공유 아이콘(↗) 클릭
아티팩트가 열린 상태에서 패널 상단에 있는 공유 아이콘 또는 "Share" 버튼을 찾습니다. 아티팩트 미리보기 탭과 코드 탭 사이 상단에 위치합니다.
공유 설정 선택
팝업이 열리면 공유 범위를 설정합니다:
• 링크 있는 모든 사람 — 링크를 가진 누구나 접근 가능
• 클로드 사용자만 — Claude 계정 로그인 필요
대부분의 경우 "링크 있는 모든 사람"으로 설정하면 편합니다.
"Publish" 또는 "링크 복사" 클릭
공유 링크가 생성됩니다. 클립보드에 자동 복사되거나 수동으로 복사할 수 있습니다. 링크 형식: claude.site/artifacts/[고유ID]
링크 테스트 — 다른 브라우저나 시크릿 창에서 열기
공유 전에 반드시 시크릿 창이나 다른 브라우저에서 링크를 열어서 의도한 대로 보이는지 확인하세요. 로그인 없이도 보이는지 확인이 중요합니다.
공유 및 수정 반복
아티팩트를 수정한 후 다시 Publish하면 같은 링크에 업데이트된 버전이 반영됩니다. 링크를 새로 만들 필요가 없습니다.

공유 설정 옵션

✅ 모바일 브라우저 완전 지원
✅ 아티팩트 수정 시 링크 유지
✅ 링크 비활성화로 공유 종료
⚠️ 코드 소스는 기본 비공개

1-2. Publish 전 체크리스트

⚠️ 공유 전 반드시 확인하세요
개인정보 확인: 아티팩트 안에 실명, 이메일, 전화번호, 내부 데이터가 포함되어 있지 않은지 확인
사내 정보 확인: 매출, 비용, 고객사명 등 기밀 정보는 더미 데이터로 교체 후 공유
저작권 확인: 외부 이미지, 폰트, 라이브러리 사용 시 라이선스 확인
동작 테스트: 시크릿 창에서 모든 기능이 의도한 대로 작동하는지 확인

📤 Publish 활용 사례 — 발표 · 포트폴리오 · 팀 공유

활용 사례 1: 발표 자료 대신 링크 공유

❌ 기존 방식
파워포인트 파일을 메일로 보냄
→ 받는 사람이 다운로드해야 함
→ 버전 관리가 안 됨
→ 모바일에서 보기 불편
→ 수정하면 다시 보내야 함
✅ Publish 방식
인터랙티브 대시보드 링크 공유
→ 브라우저에서 즉시 열림
→ 링크 하나로 항상 최신 버전
→ 모바일 완벽 지원
→ 클릭하고 조작 가능

Section 3 · Remix 기초

🎛️ Remix — 다른 아티팩트를 내 것으로 만들기

다른 사람이 만든 훌륭한 아티팩트를 발견했을 때, 처음부터 만드는 대신 그것을 출발점으로 삼아 빠르게 내 버전을 완성합니다.

Remix 방법 3가지

🔗
방법 1: 공유 링크에서 Remix
공개된 아티팩트 링크를 열면 "Remix" 버튼이 표시됩니다. 클릭하면 해당 아티팩트가 내 Claude 세션으로 복사되어 수정 가능합니다.
가장 쉬운 방법
📋
방법 2: 코드 복사 후 수정 요청
아티팩트 코드 탭에서 전체 코드를 복사하고, 새 대화창에서 "이 코드를 기반으로 [수정 내용]으로 바꿔줘"라고 요청합니다.
범용적 방법
💬
방법 3: URL 언급 후 리믹스
Claude 대화창에서 공유 링크를 직접 언급하며 "이 링크의 아티팩트를 [변경 내용]으로 수정해줘"라고 요청합니다.
직접 링크 방식

Remix 시 클로드에게 코드 전달하는 방법

리믹스의 핵심은 기존 코드를 클로드에게 정확히 전달하는 것입니다. 두 가지 방법 중 상황에 맞는 것을 선택하세요:

방법언제 사용?장점단점

# 기존 아티팩트의 데이터와 스타일을 내 것으로 교체 아래 아티팩트를 우리 팀 버전으로 리믹스해줘. [기존 코드 — 여기에 붙여넣기] --- 리믹스할 아티팩트의 코드를 전체 복사해서 여기에 붙여넣으세요 --- [변경 사항 1 — 데이터 교체] 기존 데이터를 아래 우리 팀 데이터로 교체해줘: [우리 팀의 실제 또는 더미 데이터를 입력] [변경 사항 2 — 브랜드 색상] 색상 팔레트를 우리 브랜드 색상으로 변경: 메인 컬러: [예: #4F46E5] 서브 컬러: [예: #10B981] 배경: [예: #0F172A (다크) 또는 #FFFFFF (라이트)] [변경 사항 3 — 추가 기능] [없으면 "기존 기능 유지"라고 적어도 됩니다] 전체 구조와 레이아웃은 최대한 유지하면서, 위 변경 사항만 적용해줘.

# 퀴즈 게임의 문제 세트만 교체해서 완전히 다른 주제의 게임으로 아래 퀴즈 게임의 문제 세트를 교체해서 새로운 주제의 게임으로 만들어줘. [기존 퀴즈 코드] --- 기존 퀴즈 아티팩트 코드 붙여넣기 --- [새 문제 세트 — 주제: [예: 우리 회사 제품 지식 퀴즈]] 문제 1: "[질문]" 정답: [OX 또는 4지선다 정답] 오답 설명: "[틀렸을 때 보여줄 설명]" 문제 2: "[질문]" 정답: [정답] 오답 설명: "[설명]" [계속 추가...] [추가 변경] - 게임 제목을 "[새 제목]"으로 변경 - 시작 화면 안내 문구 교체: "[새 안내 문구]" - 완료 화면 메시지: "[새 완료 메시지]" 게임 로직과 UI는 그대로 유지하고, 콘텐츠만 교체해줘.
Section 4 · Remix 6가지 전략

🎯 Remix 6가지 전략 — 심화 활용

단순한 데이터 교체를 넘어 아티팩트를 근본적으로 발전시키는 6가지 고급 리믹스 전략을 배웁니다.

🔄 전략 1 — 언어·지역화 리믹스
영어로 된 아티팩트를 한국어로, 또는 반대로 변환합니다. UI 텍스트, 날짜 형식, 통화 단위, 문화적 맥락을 현지화합니다.
🔄 전략 2 — 타겟 오디언스 리믹스
동일한 정보를 다른 독자를 위해 재구성합니다. 임원용 요약본↔실무자용 상세본, 고객용↔내부용, 초보자용↔전문가용 전환.
🔄 전략 3 — 기능 확장 리믹스
기존 아티팩트에 없는 기능을 추가합니다. 검색 기능, 필터, 내보내기, 다크모드, 저장 기능 등을 기존 코드 위에 층층이 쌓습니다.
🔄 전략 4 — 합치기(Merge) 리믹스
두 개 이상의 아티팩트를 하나로 합칩니다. 퀴즈 앱 + 리더보드, 대시보드 + 편집 폼, 게임 + 학습 자료 등 조합으로 새로운 가치를 만듭니다.
🔄 전략 5 — 플랫폼 이식 리믹스
HTML 아티팩트를 React로, 정적 페이지를 인터랙티브로, 또는 반대로 이식합니다. 동일한 기능을 다른 기술 스택으로 재구현합니다.
🔄 전략 6 — 축소(Micro) 리믹스
복잡한 아티팩트에서 핵심 기능만 추출해 가볍고 빠른 버전을 만듭니다. 전체 대시보드에서 핵심 차트 1개만, 큰 게임에서 핵심 메커니즘만 추출.
🎭 실습 시나리오 4
상황: 영어로 된 프로젝트 관리 대시보드 아티팩트를 발견했습니다. 한국어로 바꾸고 우리 회사 업무 방식에 맞게 기능을 추가하고 싶습니다.
목표: 전략 1(언어화) + 전략 3(기능 확장)을 동시에 적용한 리믹스
P-08복사해서 바로 쓰는 프롬프트 — Remix 심화: 다기능 통합 리믹스
# 여러 리믹스 전략을 동시에 적용하는 심화 요청 아래 아티팩트를 여러 방향으로 동시에 리믹스해줘. [원본 코드] --- 리믹스할 코드 붙여넣기 --- [리믹스 1 — 언어·지역화] 모든 UI 텍스트를 한국어로 번역해줘. 날짜 형식: YYYY년 MM월 DD일 숫자: 천 단위 쉼표 (1,000,000) 통화: 달러($) → 원(₩) [리믹스 2 — 기능 확장] 아래 기능 2가지를 추가해줘: ① 검색/필터 기능: 상단에 검색창, 입력하면 관련 항목만 표시 ② CSV 내보내기: "엑셀로 다운로드" 버튼 (테이블 데이터를 CSV로) [리믹스 3 — 오디언스 조정] 현재 개발자용으로 보이는 기술 용어들을 일반 직원도 이해할 수 있는 용어로 변경해줘. 예: "API 호출 횟수" → "서버 요청 수", "Latency" → "응답 속도" 원래의 기능과 디자인 방향은 최대한 유지하면서 위 변경만 적용해줘.
P-09복사해서 바로 쓰는 프롬프트 — Remix 심화: 두 아티팩트 합치기
# 두 개의 아티팩트를 하나의 통합 앱으로 합치기 두 개의 아티팩트를 합쳐서 하나의 통합 앱으로 만들어줘. [아티팩트 A — 퀴즈 게임] --- 첫 번째 아티팩트 코드 붙여넣기 --- [아티팩트 B — 점수 리더보드] --- 두 번째 아티팩트 코드 붙여넣기 --- [통합 방식] ① 상단 탭 네비게이션: "퀴즈 플레이" / "리더보드" 탭 ② 퀴즈 완료 시 점수를 자동으로 리더보드에 등록 ③ 플레이어 이름 입력 → 퀴즈 시작 → 완료 시 리더보드로 이동 ④ 리더보드는 localStorage에 저장 (새로고침해도 유지) [디자인] 두 아티팩트의 디자인 중 더 나은 것을 기준으로 통일해줘. 탭 간 전환은 부드러운 애니메이션.
Section 5 · 협업 워크플로

🤝 협업 워크플로 — 팀과 함께 아티팩트 발전시키기

Publish와 Remix를 팀 협업에 적용하면 새로운 형태의 비동기 협업이 가능합니다.

팀 협업 루프

1
기획자가 와이어프레임 아티팩트 Publish
HTML로 UI 와이어프레임 만들기 → 링크 공유 → "이 구조로 맞나요?" 피드백 수집
2
개발자가 Remix → 실제 데이터 연결
와이어프레임 코드 가져와서 → API 연결, 실제 데이터로 교체 → 개선 버전 Publish
3
디자이너가 Remix → 시각 개선
개발자 버전 가져와서 → 색상, 타이포그래피, 레이아웃 개선 → 최종 디자인 Publish
4
팀장이 Remix → 최종 검토·수정
최종 버전 가져와서 → 내용 정확도 검토, 마지막 수정 → 고객/임원에게 공유

데이터 분리: 데이터(내용)와 UI(구조)를 명확히 분리. 데이터만 바꾸면 완전히 다른 아티팩트가 되도록 설계.
변수화: 바뀔 수 있는 부분(브랜드 색상, 회사명, 데이터)을 파일 상단에 모아서 한 곳에서 관리.
주석 포함: 다음에 리믹스할 때 어디를 바꾸면 되는지 주석으로 표시.
🎭 실습 시나리오 6 (마스터 레벨)
상황: 매월 다른 데이터로 동일한 형식의 보고서 대시보드를 만들어야 합니다. 매달 처음부터 만드는 것이 비효율적입니다.
목표: 데이터만 바꾸면 되는 재사용 가능한 마스터 템플릿 만들기
P-12복사해서 바로 쓰는 프롬프트 — 마스터 템플릿: 재사용 가능한 월간 보고서
# 매달 데이터만 바꿔서 재사용하는 보고서 아티팩트 HTML 아티팩트로 매월 재사용 가능한 마스터 보고서 템플릿을 만들어줘. 다음 달에 이 코드를 리믹스할 때 데이터 부분만 바꾸면 완성되도록. [설계 방식] 파일 상단에 모든 변경 가능 데이터를 JavaScript 객체로 분리해줘: // === 여기만 수정하세요 (매달 업데이트) === const REPORT_DATA = { month: "[월 이름]", year: "[연도]", company: "[회사명]", kpis: [ { name: "[KPI명]", value: [수치], target: [목표], unit: "[단위]" }, // ... 추가 ], highlights: [ "[이번 달 주요 성과 1]", "[이번 달 주요 성과 2]", ], challenges: ["[이슈 1]"], nextMonthGoals: ["[목표 1]", "[목표 2]"] }; // === 위만 수정, 아래는 건드리지 마세요 === [보고서 UI] ① 헤더: 회사명 + 월간 보고서 + 날짜 ② KPI 카드 그리드: 각 KPI 수치 + 목표 대비 달성률 + 색상 ③ 주요 성과 목록 ④ 이슈/도전 목록 ⑤ 다음 달 목표 ⑥ 인쇄 버튼 (브라우저 인쇄 다이얼로그) 주석으로 "다음에 사용할 때 변경할 부분"을 명확히 표시해줘.
P-13복사해서 바로 쓰는 프롬프트 — 리믹스 체인: 기존 아티팩트에서 새 버전 파생
# 잘 만들어진 아티팩트를 완전히 다른 용도로 전환하기 아래 아티팩트를 완전히 다른 용도의 새 아티팩트로 변환해줘. 핵심 UI 패턴과 코드 구조는 최대한 재활용하면서. [원본 아티팩트] --- 변환할 코드 붙여넣기 --- [원본의 용도] [예: 팀 KPI 대시보드] [새 용도] [예: 개인 목표 트래커] [유지할 것] - 전체 레이아웃 구조 - 차트/시각화 유형 - 인터랙션 방식 (클릭·호버 등) [변경할 것] - 데이터 구조와 의미 (팀 KPI → 개인 목표) - 레이블과 텍스트 전체 - 색상 팔레트 (새 용도에 맞게: [새 색상 방향]) - 기능 추가/제거: [구체적으로 명시] 변환 후 "원본과 새 버전의 주요 차이점"을 3줄로 요약해줘.
