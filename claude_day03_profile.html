Section 1

왜 Profile 설정이 핵심인가?

설정 전과 후, 클로드의 답변이 어떻게 달라지는지 먼저 확인해봅니다.

설정하지 않으면 벌어지는 일

클로드는 기본 상태에서 "모든 사람을 위한 중립적인 답변"을 제공합니다. 매번 새 대화를 시작할 때마다 클로드는 당신이 누구인지, 어떤 분야에서 일하는지, 어떤 스타일을 좋아하는지 전혀 모릅니다.

결과적으로 매번 같은 설명을 반복하게 되고, 답변이 너무 길거나 너무 초보적이거나, 관련 없는 내용으로 채워지는 경험을 하게 됩니다.

❌ 설정 없이 질문했을 때
질문: "마케팅 캠페인 성과 분석해줘"

클로드 반응: "마케팅 캠페인 성과 분석이란 무엇인가요? 마케팅은 크게 디지털 마케팅과 전통적인 마케팅으로 나뉘며… KPI란 Key Performance Indicator의 약자로…"

→ 이미 아는 기초 설명부터 시작. 시간 낭비.
✅ Profile 설정 후 같은 질문
질문: "마케팅 캠페인 성과 분석해줘"

클로드 반응: "10년 경력 B2B 마케터 관점에서 분석해드릴게요. ROAS, CAC, LTV 기준으로 보면… 특히 리드 퀄리티 지표를 보셨나요?"

→ 전문가 수준에서 바로 핵심으로 진입.

Profile 설정이 만드는 3가지 변화

반복 설명이 사라진다
"나는 마케터야", "간결하게 답해줘", "한국어로 써줘" — 이런 말을 매번 새 대화마다 입력할 필요가 없어집니다. 한 번 설정하면 모든 대화에 자동 적용됩니다.
답변의 수준이 맞춰진다
전문 분야는 심화 수준으로, 새로 배우는 분야는 쉬운 설명으로 자동 조절됩니다. 내가 모르는 것은 설명하고, 이미 아는 것은 건너뜁니다.
클로드가 '나의 조력자'처럼 행동한다
내 업무 맥락, 사용하는 도구, 선호하는 포맷을 알기 때문에 질문의 숨은 의도까지 파악해 더 적합한 답을 줍니다.
💡 핵심 사실
Profile 설정은 모든 플랜(무료 포함)에서 사용 가능합니다. 설정하는 데 10분도 안 걸리지만, 매일 클로드를 쓸 때마다 그 효과를 느낄 수 있습니다. Anthropic 공식 지원센터에 따르면 Profile에 입력한 내용은 모든 대화에 자동으로 적용됩니다.
✅ 적용방법 — 먼저 현재 상태 진단하기
설정 전, 아래 프롬프트로 클로드가 현재 나를 어떻게 인식하고 있는지 확인해 보세요. 그 결과를 보면 왜 설정이 필요한지 바로 체감할 수 있습니다.
🎭 실습 시나리오 1
상황: Profile 설정을 하기 전 현재 상태입니다.
목표: 클로드가 나를 얼마나 알고 있는지 확인하고, 설정의 필요성을 체감하기
P-01복사해서 바로 쓰는 프롬프트 — 현재 인식 진단
지금 네가 나에 대해 알고 있는 정보를 모두 말해줘. ① 내 직업이나 역할 ② 내가 선호하는 답변 스타일 ③ 내가 자주 쓰는 언어나 용어 ④ 내 전문 분야와 비전문 분야 솔직하게 "모른다"면 모른다고 해줘. 그리고 이 정보들을 알면 어떤 점이 더 나아질지도 알려줘.
P-02복사해서 바로 쓰는 프롬프트 — 설정 필요성 체험
나는 [내 직업]이고, 주로 [주요 업무]를 해. 지금 이 정보 없이 받은 답변과, 이 정보를 알고 받을 답변이 어떻게 달라지는지 비교해줄 수 있어? 예시 질문: "[나의 실제 업무 질문 하나]" ① 내 배경 모르는 상태에서의 답변 ② 내 배경 아는 상태에서의 답변 두 가지 버전으로 보여줘.
Section 2

클로드 3-레이어 개인화 구조 이해하기

Profile 설정을 효과적으로 활용하려면 클로드 개인화의 3단계 구조를 먼저 이해해야 합니다.

3개 레이어가 쌓이는 방식

🧍 레이어 1: Profile Preferences (개인 프로필) ← 오늘의 주제
모든 대화에 항상 적용 · 내가 누구인지, 어떻게 일하는지 · 계정 전체에 영구 적용
위치: Settings → Profile → "What personal preferences should Claude consider in responses?"
↓ 그 위에 추가로
📁 레이어 2: Project Instructions (프로젝트 지침)
특정 프로젝트 대화에만 적용 · "이 Next.js 프로젝트는 TypeScript 사용" 같은 프로젝트 특화 규칙 · 무료 플랜도 프로젝트 5개까지 가능
↓ 그 위에 추가로
🎨 레이어 3: Styles (응답 스타일)
말투·포맷·길이 조절 · 대화 중 언제든 전환 가능 · Formal / Concise / Explanatory / 커스텀
내 글 샘플 업로드 시 내 문체 그대로 모방 가능
💡 레이어 활용 전략
Profile에는 "나라는 사람"에 대한 보편적 정보만 넣으세요. 예: 직업, 경험 수준, 언어 선호.
프로젝트에는 그 프로젝트 특유의 맥락만. 예: "이 앱은 React + Supabase 사용".
Styles는 상황에 따라 그때그때 전환. 예: 보고서 작성 시 Formal, 브레인스토밍 시 Concise.

Profile에 프로젝트 전용 규칙을 넣지 마세요. 모든 대화에 불필요한 정보가 따라다니게 됩니다.

Profile 설정 화면 열기 · 항목별 작성법

실제로 설정 화면을 열고 각 항목에 무엇을 어떻게 쓰는지 배웁니다.

3-1. 설정 화면 여는 방법 (단계별)

화면 왼쪽 맨 아래 — 프로필 아이콘 찾기
claude.ai 좌측 사이드바 가장 아래에 있는 내 이니셜 원형 아이콘(예: 김이면 "김", 영문명이면 "K")을 클릭합니다. 모바일 앱이라면 오른쪽 상단 ≡ 메뉴 버튼을 누르세요.
팝업 메뉴에서 "Settings" 클릭
이니셜 아이콘을 누르면 작은 팝업이 뜹니다. 여기서 Settings (설정)을 선택합니다. 한국어로 설정된 경우 "설정"으로 표시될 수 있습니다.
Settings 화면에서 "Profile" 탭 확인
Settings 화면이 열리면 여러 탭(Profile, Appearance, Privacy 등)이 보입니다. Profile 탭을 클릭하세요. 기본으로 Profile이 선택되어 있을 수도 있습니다.

3-2. Job Function(직업 카테고리) 드롭다운

설정 화면 상단에 Job Function이라는 드롭다운 메뉴가 있습니다. 여기서 본인의 직군에 가장 가까운 카테고리를 선택하세요. 이 선택만으로도 클로드가 기본 맥락을 인식하기 시작합니다.

주요 선택지: Marketing · Engineering/IT · Sales · Education · Finance · Healthcare · Legal · Creative · Research · Management · Other

💡 TIP — 완벽한 카테고리가 없을 때
딱 맞는 카테고리가 없다면 가장 가까운 것을 선택한 뒤, 아래 텍스트 박스에 "나는 OOO 분야에서 일하며, 구체적으로는 XXX를 합니다"라고 추가로 설명하면 됩니다.

3-3. 개인 설정 텍스트 박스 — 5가지 핵심 항목

이 텍스트 박스는 자유 형식입니다. 하지만 아무거나 적으면 효과가 없습니다. 아래 5가지 항목을 포함해야 클로드가 실질적으로 달라집니다.

항목설명예시
① 나는 누구인가 직업, 역할, 경력 수준 "나는 8년 경력 UX 디자이너야"
② 전문 vs 비전문 잘 아는 분야 / 배우는 중인 분야 "피그마는 전문가 수준, 코딩은 초보"
③ 주요 업무 클로드에게 자주 부탁할 작업 "주로 사용자 리서치 분석, 와이어프레임 기획서 작성"
④ 답변 스타일 길이, 포맷, 언어, 톤 선호도 "짧고 핵심만, 한국어, 예시 꼭 포함"
⑤ 행동 규칙 해줬으면 하는 것 / 하지 말았으면 하는 것 "모르면 솔직히 말해줘, 과한 칭찬은 빼줘"
⚠️ 주의 — 이것만은 피하세요
너무 길게 쓰지 마세요. Profile 설정 내용은 모든 대화에서 토큰을 소비합니다. 500단어(한국어 약 250줄)를 넘기면 오히려 실제 대화에 쓸 수 있는 공간이 줄어듭니다.

프로젝트 전용 정보는 넣지 마세요. "내 현재 앱은 React 18을 씁니다" 같은 내용은 모든 대화에서 불필요하게 따라다닙니다. 해당 프로젝트 Instructions에만 넣으세요.
✅ 적용방법 — AI가 대신 설정문 초안 만들어주기
빈 칸 앞에서 막막하다면 걱정 마세요. 클로드에게 질문에 답하면서 자동으로 설정 초안을 만들어달라고 하면 됩니다.
🎭 실습 시나리오 2
상황: Profile 텍스트 박스 앞에서 뭘 써야 할지 모릅니다.
목표: 클로드와 대화하면서 나만의 Profile 설정 초안 완성하기
P-04복사해서 바로 쓰는 프롬프트 — 인터뷰 방식으로 Profile 초안 만들기
내 클로드 Profile 설정문을 만들어줘. 나에게 질문을 5개 던져서 답변을 받은 뒤, 그 내용을 Profile에 바로 넣을 수 있는 형식으로 정리해줘. 질문은 이 순서로 해줘: 1. 직업과 경력 수준 2. 전문 분야와 배우고 싶은 분야 3. 클로드를 주로 어떤 용도로 쓸 건지 4. 선호하는 답변 스타일 (길이, 언어, 포맷) 5. 클로드에게 반드시 해줬으면 하는 것 / 절대 하지 말았으면 하는 것 한 번에 하나씩 물어봐줘. 내가 다 답하면 최종 Profile 설정 텍스트로 정리해줘.
P-05복사해서 바로 쓰는 프롬프트 — 내 정보 한 번에 입력해서 Profile 생성
아래 내 정보를 바탕으로 클로드 Profile 설정에 바로 붙여넣을 수 있는 텍스트를 만들어줘. 300자 이내로, 핵심만 담아서. [내 정보] 직업/역할: [예: 중학교 과학 교사 / 10년 경력] 전문 분야: [예: 생물학, 화학 실험 설계] 비전문 분야: [예: 코딩, 재무 분석] 주요 사용 목적: [예: 수업 자료 제작, 학생 질문 답변 준비] 선호 언어: 한국어 선호 스타일: [예: 간결하게, 예시 위주, 단계별로] 하지 말았으면 하는 것: [예: 너무 긴 서론, 이미 아는 기초 설명] 결과물은 자연스러운 문장 형태로, 내가 Settings에 바로 붙여넣을 수 있게 해줘.

Section 5

설정 후 테스트 · 지속적 개선하기

설정을 저장했다고 끝이 아닙니다. 테스트하고 다듬는 과정이 진짜 시작입니다.

5-1. 설정 적용 여부 확인하는 방법

Settings에 저장한 뒤, 새 대화창을 열어서 아래 프롬프트로 확인하세요. 기존 대화에서는 이미 시작한 이후라 설정이 완전히 반영되지 않을 수 있습니다.

🎭 실습 시나리오 3
상황: Profile 설정을 저장하고 새 대화창을 열었습니다.
목표: 설정이 제대로 적용되었는지 확인하고, 실제 업무 시나리오로 테스트하기
P-06복사해서 바로 쓰는 프롬프트 — 설정 적용 확인 테스트
내 Profile 설정이 잘 반영되고 있는지 확인하고 싶어. ① 지금 네가 파악하고 있는 나의 정보를 요약해줘 (직업, 전문 분야, 선호 스타일, 행동 규칙) ② 방금 요약한 내용을 바탕으로, 아래 질문에 답해줘: "[내 실제 업무와 관련된 구체적인 질문 하나]" ③ 이 설정에서 빠진 게 있거나, 추가하면 더 좋아질 정보가 있으면 제안해줘
P-07복사해서 바로 쓰는 프롬프트 — 3가지 업무 시나리오 동시 테스트
내 Profile 설정이 실제 업무에 잘 맞는지 테스트해줘. 아래 3가지 미니 과제를 내 배경에 맞춰서 처리해줘. 과제 1 (분석): [예: 최근 SNS 알고리즘 변화가 내 업무에 미칠 영향 3가지] 과제 2 (작성): [예: 팀원에게 보낼 이번 주 업무 현황 요약 이메일 초안] 과제 3 (아이디어): [예: 다음 달 캠페인 아이디어 3가지, 각각 한 줄 설명] 처리 후 마지막에 "이 설정으로 더 잘 도울 수 있도록 추가하면 좋을 정보"도 한 줄 제안해줘.

5-2. 설정 진화시키기 — 언제 업데이트해야 할까?

클로드가 계속 같은 실수를 반복할 때
예: 매번 너무 긴 답변, 자꾸 영어로만 답하기, 기초부터 설명하기 → 해당 규칙을 "하지 말 것" 목록에 추가
직업이나 역할이 바뀔 때
이직, 승진, 새 프로젝트 시작, 사이드 프로젝트 추가 등 → 직업 정보 업데이트
새 도구나 기술을 배우기 시작할 때
새로 공부하는 언어, 툴, 분야를 "배우는 중" 섹션에 추가 → 자동으로 쉬운 설명 모드
3~4주마다 정기 점검
"이 설정이 여전히 나에게 맞나?" 스스로 점검. 불필요한 규칙은 삭제, 새로운 니즈는 추가.

5-3. Memory(메모리) 기능과 Profile의 조합

2026년 3월부터 클로드에는 메모리 기능이 무료를 포함한 모든 플랜에 제공됩니다. Profile 설정과 Memory는 역할이 다르게 작동합니다.

구분Profile 설정Memory (메모리)
무엇을 기억하나내가 직접 쓴 고정 정보대화에서 자동으로 추출한 정보
업데이트 방식내가 직접 Settings에서 수정자동 (내가 편집·삭제 가능)
적합한 정보변하지 않는 핵심 정보 (직업, 스타일)진행 중 프로젝트, 개인 취향 등 유동적 정보
확인 위치Settings → ProfileSettings → Memory
💡 조합 전략 — Profile + Memory 함께 쓰기
Profile: 변하지 않는 핵심 — 직업, 전문 분야, 언어 선호, 행동 규칙
Memory: 클로드가 대화 중 자동으로 파악한 것들 — 내가 좋아하는 커피 브랜드, 진행 중인 프로젝트명, 내 글쓰기 버릇 등

두 가지가 함께 작동하면, 시간이 지날수록 클로드는 마치 오래 함께 일한 동료처럼 나를 파악합니다.
✅ 적용방법 — 지금 당장 Memory에 핵심 정보 심어두기
대화를 통해 클로드의 Memory에 중요한 정보를 직접 학습시킬 수 있습니다. 아래 프롬프트를 사용하면 Memory에 저장될 수 있는 핵심 정보를 한 번에 전달할 수 있습니다.
🎭 실습 시나리오 4
상황: Profile 설정은 완료했고, Memory 기능도 켜져 있습니다.
목표: 앞으로의 대화에서 클로드가 더 잘 나를 이해할 수 있도록 핵심 정보를 Memory에 등록하기
P-08복사해서 바로 쓰는 프롬프트 — Memory에 핵심 정보 등록하기
앞으로 나와 더 잘 협업할 수 있도록, 내 정보를 기억해줘. [기억해줬으면 하는 것들] 1. 현재 진행 중인 주요 프로젝트: [예: OO앱 리뉴얼 기획, 출시 목표 6월] 2. 자주 사용하는 도구: [예: Notion, Figma, Slack, Google Analytics] 3. 나의 업무 스타일: [예: 완벽주의보다 빠른 실행 선호, 1일 1배포 지향] 4. 내가 싫어하는 것: [예: 불필요한 회의, 모호한 피드백] 5. 내가 지금 공부하는 것: [예: SQL 데이터 분석, 영어 비즈니스 글쓰기] 이 내용을 메모리에 저장하고, 앞으로 관련 대화를 할 때 참고해줘. 기억한 내용을 요약해서 확인시켜줘.
P-09복사해서 바로 쓰는 프롬프트 — Profile 설정 한 달 후 재검토
내 Profile 설정을 재검토하고 싶어. ① 지금까지 우리가 나눈 대화 패턴을 보면, 내 설정에서 실제로 잘 작동한 부분은 뭐야? ② 반대로 설정했지만 실제 대화에서 지켜지지 않았던 부분이 있어? ③ 새로 추가하면 좋을 규칙이 보이는 게 있어? ④ 지금 내 Profile 설정이 500자를 넘는다면, 어떤 항목을 줄이거나 합치면 좋을지 제안해줘.

5-4. 직업별 맞춤 후속 활용 — 설정 완료 후 첫 업무 테스트

🎭 종합 실습 시나리오 (마무리)
상황: Profile 설정 완료, 새 대화창을 열었습니다.
목표: 설정된 나의 배경을 바탕으로 클로드와 실제 업무를 처리해보기
P-10복사해서 바로 쓰는 프롬프트 — 오늘의 업무 협업 시작
오늘 내가 처리해야 할 업무를 함께 정리해줘. [오늘의 할 일] 1. [업무 1] 2. [업무 2] 3. [업무 3] 4. [업무 4] ① 내 직업과 현재 상황을 고려해서 우선순위를 재정렬해줘 ② 클로드가 직접 도울 수 있는 것과, 내가 직접 해야 하는 것을 분류해줘 ③ 클로드가 도울 수 있는 것 중 가장 빠르게 처리할 수 있는 것부터 시작하자 준비됐으면 첫 번째 업무부터 바로 시작해줘.
P-11복사해서 바로 쓰는 프롬프트 — 클로드에게 내 업무 스타일 가르치기
내가 좋아하는 결과물 스타일을 직접 보여줄게. 이 샘플을 보고 앞으로 내가 비슷한 요청을 할 때 이 스타일을 따라줘. [샘플 — 내가 직접 쓴 문서/이메일/코드 중 하나 붙여넣기] --- 여기에 내가 직접 쓴 글, 이메일, 보고서 중 하나를 붙여넣으세요 --- 이 샘플에서 파악할 수 있는 내 글쓰기 특징: ① 문장 길이 ② 용어 선택 스타일 ③ 구조 잡는 방식 ④ 특이한 표현 습관 이 특징들을 분석해서 알려줘. 앞으로 내가 글 작성을 부탁할 때 이 스타일을 유지해줘.
P-12복사해서 바로 쓰는 프롬프트 — 나만의 맞춤 Profile 최종 버전 완성
지금까지 우리가 나눈 대화들을 바탕으로, 내 클로드 Profile 설정의 최종 버전을 만들어줘. 조건: - 300자 이내 (토큰 절약) - 실제로 행동 변화를 일으키는 규칙만 포함 ("좋은 질문이에요" 같은 것만 빼도 되는 건 제거) - "나는 누구인가 + 전문/비전문 + 주요 용도 + 스타일 + 금지 규칙" 5가지 포함 - 내가 Settings에 바로 붙여넣을 수 있는 형식으로 최종 결과물 아래에, 이 설정으로 기대되는 효과 3가지도 간략히 적어줘.
