좋은 다단계 프롬프트의 5가지 설계 원칙

1

단계를 명시적으로 번호 매기기

"다음을 순서대로 해줘: 1단계... 2단계..."처럼 순서를 명확히 하면 중간에 길을 잃지 않습니다. 단계 완료 보고도 요청하세요.

2

각 단계의 성공 조건 정의

"로그인이 완료된 것은 대시보드 화면이 보일 때"처럼 각 단계가 언제 완료됐는지 클로드가 알 수 있도록 합니다.

3

실패 시 처리 방법 명시

"페이지를 찾지 못하면 다음 사이트로 넘어가줘", "오류가 나면 스크린샷과 함께 나에게 보고해줘"처럼 예외 상황을 미리 정의합니다.

4

핵심 체크포인트 설정

되돌릴 수 없는 액션(파일 저장, 전송, 삭제) 전에는 반드시 확인을 받도록 설계합니다. "저장 전 최종 결과물을 보여주고 확인 후 저장해줘."

5

중간 결과물 저장 요청

긴 파이프라인에서 중간 데이터를 임시 파일로 저장해두면, 후반부에 실패해도 처음부터 다시 하지 않아도 됩니다.

💡 오늘 실습 환경 선택 Claude Cowork 사용자: L1, L2의 프롬프트를 그대로 사용. Claude Cowork 세션에서 실행하면 됩니다.
Claude Code Desktop 사용자: L3의 Claude Code 전용 프롬프트를 사용. computer-use MCP가 활성화된 상태에서 실행.
두 환경이 없다면 프롬프트 구조와 패턴을 익히고, 직접 적용 가능한 상황을 파악하는 데 집중하세요.
L1

브라우저 자동화 — 검색·수집·정리를 한 번에

⏱ 예상 시간 30분  |  Claude Cowork + Chrome 확장

업무 시간의 상당 부분은 검색하고, 읽고, 정리하는 반복입니다. 클로드가 브라우저를 조작해 이 과정을 자동화하면, 여러분은 수집된 인사이트에만 집중할 수 있습니다.

적용 방법 | 브라우저 자동화 워크플로우 설계
  1. 1
    목표 사이트와 수집 항목을 명확히 정의하라 "웹에서 찾아줘"보다 "네이버 쇼핑에서 [제품명]을 검색하고, 가격·별점·리뷰 수를 상위 5개 제품에서 추출해줘"처럼 구체적으로 지정합니다. 클로드가 헤매지 않습니다.
  2. 2
    페이지네이션과 로딩 시간을 고려하라 "여러 페이지가 있으면 최대 3페이지까지 수집해줘", "동적 콘텐츠가 로딩될 때까지 기다려줘"처럼 페이지 구조를 안내합니다. 특히 자바스크립트 렌더링 사이트는 로딩 대기가 필요합니다.
  3. 3
    수집 완료 후 즉시 파일로 저장 요청 브라우저 탭이 닫히거나 세션이 끊기면 수집한 데이터가 사라질 수 있습니다. 수집과 동시에 로컬 파일로 저장하도록 지시합니다.

L2

데스크탑 앱 조작 — 코드 없이 앱을 제어하다

⏱ 예상 시간 30분  |  GUI 앱 직접 자동화

API가 없거나, GUI에서만 할 수 있는 작업들이 있습니다. 클로드는 화면을 보고 어떤 앱이든 인간처럼 조작할 수 있습니다. 엑셀, 파워포인트, 사내 전용 소프트웨어까지 포함해서요.

⚠️ 앱 자동화 시 주의사항 앱을 직접 조작하므로 예상치 못한 변경이 생길 수 있습니다. 중요한 파일은 미리 복사본(백업)을 만들고 시작하세요. 특히 저장, 전송, 삭제 작업은 반드시 확인 단계를 넣으세요.
적용 방법 | 데스크탑 앱 자동화 핵심 패턴
  1. 1
    앱 이름과 버전을 명시하라 "엑셀로 만들어줘"보다 "Microsoft Excel을 열고"처럼 정확한 앱 이름을 쓰면 클로드가 올바른 앱을 찾습니다. 버전별로 UI가 다를 경우 버전도 명시하세요.
  2. 2
    메뉴 경로를 구체적으로 설명하라 "차트를 만들어줘"보다 "삽입 → 차트 → 막대형 → 클러스터형 막대 선택"처럼 메뉴 경로를 알고 있다면 명시합니다. 모른다면 "차트 삽입 방법을 찾아서 진행해줘"라고 해도 됩니다.
  3. 3
    작업 전 파일 상태를 확인시켜라 "현재 파일의 내용을 먼저 스크린샷으로 보여줘"라고 하면 클로드가 올바른 파일을 열었는지 확인할 수 있습니다. 엉뚱한 파일을 수정하는 실수를 방지합니다.

⏱ 예상 시간 40분  |  여러 앱을 이어가는 실전 워크플로우

이제 진짜 실력을 발휘할 시간입니다. 브라우저 → 로컬 앱 → 파일 저장 → 이메일 발송까지 이어지는 완전한 업무 자동화 파이프라인을 실행합니다. 이것이 Computer Use의 진정한 가치입니다.

적용 방법 | 안전한 다단계 파이프라인 설계
  1. 1
    단계별 중간 저장 포인트를 의무화하라 10단계짜리 파이프라인에서 9단계에서 실패하면 처음부터 다시 해야 합니다. 3~4단계마다 중간 결과를 파일로 저장해두면 실패 지점부터 재시작이 가능합니다.
  2. 2
    단계 간 데이터 전달 방식을 명시하라 "1단계에서 수집한 URL 목록을 /tmp/urls.txt에 저장하고, 2단계에서 이 파일을 읽어서 진행해줘"처럼 단계 간 데이터 전달 경로를 명확히 합니다.
  3. 3
    최종 결과물 전 사람의 확인을 넣어라 이메일 발송, 파일 업로드, SNS 게시처럼 외부로 나가는 모든 액션은 반드시 사람이 검토한 후 실행합니다. 자동화된 실수는 되돌리기 어렵습니다.
  4. 4
    실패 시 재시작 지점을 명시하라 "2단계에서 실패하면 1단계 결과물(/tmp/urls.txt)을 유지하고 2단계만 다시 실행해줘"처럼 재시작 포인트를 정의하면 부분 재실행이 가능합니다.

📌 26일차 핵심 정리

다단계 파이프라인의 핵심

단계 번호 매기기 + 성공 조건 정의 + 실패 처리 + 체크포인트 + 중간 저장. 이 5가지가 안전하고 완성도 높은 자동화를 만든다.

브라우저 자동화

경쟁사 조사, 뉴스 수집, 폼 자동 작성. "결과 중심"으로 지시하고 데이터는 즉시 로컬 파일로 저장. 페이지네이션과 로딩 대기 명시.

앱 조작 자동화

엑셀 차트 생성, PPT 보고서 완성, 이미지 일괄 처리. 작업 전 파일 상태 확인, 저장 전 반드시 미리보기 요청.

크로스 앱 파이프라인

이메일 → 스프레드시트 → 슬랙처럼 여러 앱을 이어가는 워크플로우. 외부 전송(발송/업로드) 전 반드시 사람 확인.

재시작 & 디버깅

중간 저장 파일로 실패 지점부터 재시작. 예상치 못한 화면이나 오류는 즉시 보고 후 대기. 클릭하지 말고 확인 먼저.
