# Cowork 모드를 켜고 아래 태스크를 입력하세요 (Claude Desktop App에서만 작동) 내 바탕화면에 있는 파일들을 정리해줘. 구체적으로: ① 현재 바탕화면에 있는 파일 목록 먼저 보여줘 ② 파일 유형별로 분류 제안해줘 (문서/이미지/기타) ③ 내가 승인하면 해당 폴더를 만들고 이동해줘 시작하기 전에 작업 계획을 먼저 보여주고, 내가 "진행해줘"라고 하면 실행해줘.
Section 1 · 설치 & 첫 셋업

🖥️ Cowork 설치 & 첫 셋업 — 10분이면 시작 가능

Claude Desktop 앱 설치부터 Cowork 첫 태스크 실행까지, 단계별로 따라 합니다.

설치 단계별 가이드

Claude Desktop 앱 다운로드
claude.com/download에서 macOS 또는 Windows 버전을 다운로드합니다. 2026년 2월부터 Windows도 완전 지원합니다. (Pro 이상 플랜 필요)
앱 설치 후 로그인
claude.ai 계정으로 로그인합니다. 처음 실행 시 "Cowork Readiness Check"가 자동으로 실행되어 컴퓨터 호환성을 확인합니다.
Cowork 탭 선택
앱 상단의 모드 선택에서 "Cowork" 탭을 클릭합니다. "New Chat" 대신 "New Task" 버튼이 보이면 Cowork 모드가 활성화된 것입니다.
파일 접근 권한 설정
Settings → Cowork → File Access에서 Claude가 접근할 수 있는 폴더를 지정합니다. 최소한 작업 폴더 1개를 허용해야 파일 작업이 가능합니다. 전체 드라이브보다 특정 폴더만 허용하는 것이 안전합니다.
첫 태스크 실행 & 안전 확인
중요: Cowork는 실제 파일을 수정합니다. 처음에는 "계획 먼저 보여줘, 승인 후 실행"을 습관화하세요. 작업 취소는 어려울 수 있으니 중요 파일은 백업 후 시작하세요.

Cowork 태스크 작성의 3원칙

❌ 나쁜 Cowork 태스크 작성
"보고서 만들어줘"

→ 어디 있는 데이터를? 어떤 형식으로? 어디에 저장? 범위가 없어서 예측 불가한 결과 발생 가능
✅ 좋은 Cowork 태스크 작성
"~/Documents/data/ 폴더의 이번 주 CSV 파일들을 읽어서, 매출 현황 요약 보고서를 ~/Reports/weekly_YYYY-MM-DD.docx로 저장해줘. 시작 전에 계획 보여줘."

→ 입력·출력·위치·형식이 명확. 예측 가능.
🎭 실습 시나리오 1
상황: Cowork를 처음 설치했습니다. 내 문서 폴더에 있는 최근 3개월 보고서 파일들을 정리하고 목차를 만드는 첫 태스크를 실행해보고 싶습니다.
목표: Cowork의 파일 접근 + 분석 + 생성 3가지 능력을 한 번에 체험
P-01복사해서 바로 쓰는 프롬프트 — Cowork 첫 파일 작업 태스크
# Cowork 모드에서 입력 — 실제 폴더 경로로 수정하세요 내 문서 폴더를 분석하고 정리를 도와줘. [대상 폴더] ~/Documents/보고서/ (또는 실제 경로 입력) [요청 내용] ① 폴더 안에 있는 파일 목록을 읽어서 아래 형식으로 정리해줘: - 파일명 / 생성일 / 파일 크기 / 예상 내용 (파일명 기준) ② 분석 후 파일들을 유형별로 분류하는 방안을 제안해줘: 예) 월별 폴더, 프로젝트별 폴더, 카테고리별 폴더 ③ 내가 선택하면 실제로 폴더를 만들고 파일을 이동해줘 [안전 옵션] - 파일을 이동하기 전에 반드시 계획 목록을 보여주고 내 승인을 받아줘 - 원본 파일은 삭제하지 말고 이동만 해줘 시작 전에 "어떤 파일에 접근할지" 먼저 알려줘.
P-02복사해서 바로 쓰는 프롬프트 — 문서에서 보고서 자동 생성 (Cowork)
# 여러 소스 파일을 읽어서 하나의 완성된 보고서로 합치기 아래 파일들을 읽어서 통합 주간 보고서를 만들어줘. [읽을 파일들] ~/Work/this_week/ 폴더 안의 모든 파일 (또는 구체적 파일 경로 지정) [보고서 요구사항] 형식: 마크다운 (.md) 또는 Word (.docx) 저장 위치: ~/Reports/weekly_[오늘날짜].md 보고서 구성: ① 주간 핵심 요약 (3~5줄) ② 완료된 작업 목록 ③ 진행 중인 작업 및 상태 ④ 다음 주 계획 ⑤ 이슈/리스크 [스타일] 읽기 쉽게 제목과 불릿 포인트 사용 한국어로 작성 작업 계획을 먼저 보여주고, 내가 OK하면 실행해줘.
Section 2 · Cowork Projects

📁 Cowork Projects — 지속 가능한 작업 공간

2026년 3월에 추가된 Cowork Projects는 단순 태스크를 넘어, 로컬 폴더와 연결된 지속 가능한 작업 공간입니다. 컨텍스트가 세션 간에도 유지됩니다.

Cowork Projects vs claude.ai Projects — 차이점

구분claude.ai Projects (Day 11)Cowork Projects (오늘)
실행 환경웹 브라우저데스크탑 앱
파일 접근수동 업로드만로컬 폴더 직접 연결
작업 방식Q&A 대화형자율 에이전트 실행
컨텍스트대화 내 유지세션 간 영구 유지 (메모리)
팀 공유지원 (Team 플랜)현재 로컬 전용 (추후 지원 예정)
스케줄 실행불가가능 (정기 자동 실행)
적합한 용도팀 공유 지식 기반개인 반복 업무 자동화
🔑 두 Projects를 함께 쓰는 전략
claude.ai Projects: 팀원들이 공유할 지식 기반, 공통 Instructions, 정책 문서 보관
Cowork Projects: 개인 작업 자동화, 로컬 파일 연결, 반복 태스크 실행
→ 팀 Projects에서 공통 가이드라인을 가져와 Cowork Projects에서 실행하는 방식이 가장 효율적입니다.

Cowork Project 만들기 3가지 방법

처음부터 만들기 (Scratch)
Cowork → Projects → + → "Start from scratch" 선택 → 이름 설정 → Instructions 작성 → 연결할 로컬 폴더 선택
claude.ai Project에서 가져오기 (Import)
기존 claude.ai Projects의 Instructions와 Knowledge를 Cowork로 가져옵니다. 팀 Knowledge를 그대로 유지하면서 Cowork의 실행 능력을 추가합니다.
기존 폴더 연결 (From Folder)
이미 작업 폴더가 있다면 그 폴더를 선택해서 Project로 만듭니다. 가장 빠른 방법 — 기존 파일과 맥락이 자동으로 연결됩니다.
🎭 실습 시나리오 2
상황: 매주 동일한 방식으로 영업 보고서를 작성해야 합니다. 매번 처음부터 설명하지 않고, Cowork Project에 모든 맥락을 저장해서 "이번 주 데이터로 보고서 만들어줘" 한 마디로 실행되게 하고 싶습니다.
목표: 영업 보고서 자동화 Cowork Project 완성
P-03복사해서 바로 쓰는 프롬프트 — Cowork Project 첫 셋업 확인
# 새 Cowork Project를 만든 직후, 이 프롬프트로 Project를 "학습"시키세요 이 Project의 모든 파일과 폴더를 읽어서 나에 대해 파악해줘. [읽어야 할 것] ① 이 폴더의 모든 파일 목록 (서브폴더 포함) ② 각 파일의 역할 추론 (파일명과 최근 수정일 기준) ③ 가장 자주 사용할 것 같은 파일 타입과 패턴 [보고해줄 것] 다음 형식으로 Project 이해 보고서 작성: 📁 Project 개요 - 총 파일 수, 주요 파일 타입 - 가장 최근 수정된 파일 5개 🎯 이 Project의 주요 용도 (내 추정) 📋 내가 자주 할 것 같은 요청 5가지 (예측) ⚠️ 불명확하거나 추가 정보 필요한 것 보고서 완성 후, 내가 맞는지 틀린지 피드백 줄게.
P-04복사해서 바로 쓰는 프롬프트 — 주간 보고서 반복 태스크 (Project 맥락 활용)
# Cowork Project 내에서 매주 사용하는 보고서 생성 태스크 이번 주 영업 보고서 만들어줘. (저번 주 것을 기반으로 형식 유지해줘) [이번 주 신규 데이터] ~/Sales/this_week/ 폴더 안의 파일들 사용 [보고서 생성 조건] ① 지난 주 보고서와 동일한 형식 사용 (Project 폴더에 있는 가장 최근 weekly_report_*.docx 참조) ② 이번 주 수치로 업데이트 ③ 전주 대비 증감 자동 계산 및 강조 ④ 저장: ~/Sales/Reports/weekly_report_[오늘날짜].docx [추가 기억 사항] - 우리 팀은 매출을 "억 원" 단위로 표기 - KPI 색상: 목표 달성=초록, 미달=빨강, ±5%=노랑 - 팀장님은 요약을 맨 위 1페이지에 원함 시작하기 전에 사용할 데이터 파일 목록을 보여줘.
Section 3 · 팀 플랜 협업

👥 팀 플랜 — 공유 Projects & 권한 관리

팀 플랜에서는 여러 팀원이 같은 Projects와 Claude를 공유할 수 있습니다. 관리자 설정부터 팀원 권한까지 설정하는 방법을 배웁니다.

Team Plan 핵심 기능

기능설명관리자 설정 위치
공유 Projects팀 전체가 같은 KB·Instructions 사용claude.ai → Projects → Share
역할 기반 권한부서별·역할별 기능 접근 제한Admin Console → Roles
그룹 관리팀원을 그룹으로 묶어 일괄 설정Admin Console → Groups

팀 공유 Project 만들기 — 단계별

claude.ai에서 Project 생성 (팀장 또는 관리자)
claude.ai → Projects → New Project → 이름·목적 설정 → Instructions 작성 → Knowledge 파일 업로드
팀원에게 공유
Project 설정 → Share → 팀원 이메일 초대 또는 "팀 전체 공유" 선택. Team 플랜에서는 모든 멤버가 같은 Project에 접근 가능합니다.
역할별 Instructions 세분화
