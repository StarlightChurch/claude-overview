📖 핵심 개념

MCP란? — "클로드의 USB-C 포트"

지금까지 클로드는 인터넷과 단절된 섬이었습니다. 여러분이 직접 텍스트를 복사해서 붙여넣지 않는 한, 클로드는 여러분의 파일, 데이터베이스, 업무 도구를 전혀 알 수 없었죠.

MCP(Model Context Protocol)는 Anthropic이 2024년 공개한 오픈 표준으로, AI를 외부 데이터 소스와 도구에 연결하는 USB-C 같은 범용 인터페이스입니다. MCP가 있으면 클로드는 여러분의 하드 드라이브, 클라우드 스토리지, 데이터베이스, 업무 앱에 직접 접근할 수 있습니다.

😓
MCP 없을 때
파일 내용을 일일이 복사해서 붙여넣어야 함. 파일 10개를 분석하려면 10번 반복. 데이터가 변경되면 다시 붙여넣기 필요.
🔌
MCP 연결 후
"다운로드 폴더의 PDF 파일들을 모두 분석해줘"라고 말하면 클로드가 직접 접근. 실시간 데이터 반영.
🚀
미래의 가능성
파일시스템 + 이메일 + 캘린더 + DB를 동시에 연결하면 진정한 AI 비서 시스템 완성.

L1

환경 설정 — Claude Desktop + Node.js 준비

⏱ 예상 시간 20분  |  설치 한 번으로 끝

MCP를 쓰기 위한 두 가지 준비물: Claude Desktop 앱과 Node.js입니다. Claude Desktop은 MCP 서버를 실행하는 호스트 역할을 하고, Node.js는 MCP 서버 패키지(npx 명령어)를 실행하는 데 필요합니다.

적용 방법 | 필수 환경 설치 (단계별)
  1. 1
    Claude Desktop 다운로드 및 설치

    claude.ai/download 접속 → 운영체제에 맞는 버전 다운로드 → 설치 후 로그인. 이미 설치된 경우 최신 버전인지 확인 (메뉴 → Help → Check for Updates).

  2. 2
    Node.js 설치 확인

    터미널(macOS) 또는 명령 프롬프트(Windows)를 열고 아래 명령어를 실행합니다. 버전 번호가 나타나면 이미 설치된 것입니다. 없으면 nodejs.org에서 LTS 버전을 설치하세요.

  3. 3
    설정 파일 위치 파악

    MCP 서버 설정은 claude_desktop_config.json 파일에서 관리합니다. 아래 경로에 있습니다.

  4. 4
    Desktop Extensions 활용 (쉬운 방법)

    Claude Desktop → Settings → Extensions → "Browse extensions" 클릭. Anthropic이 검토한 공식 확장들을 클릭 한 번으로 설치 가능합니다. JSON 설정 파일 편집 없이도 많은 MCP를 설치할 수 있는 새로운 방법입니다.

# 터미널 열기: Cmd + Space → "Terminal" 검색 # Node.js 버전 확인 node --version # v20.xx.x 이상 나오면 OK # npm 확인 npm --version # 버전 번호 나오면 OK # Claude Desktop 설정 파일 위치 ~/Library/Application Support/Claude/claude_desktop_config.json # 파일이 없으면 직접 생성 mkdir -p ~/Library/Application\ Support/Claude touch ~/Library/Application\ Support/Claude/claude_desktop_config.json
:: 명령 프롬프트 열기: Win+R → "cmd" → Enter :: Node.js 버전 확인 node --version :: v20.xx.x 이상 나오면 OK :: Claude Desktop 설정 파일 위치 C:\Users\[사용자명]\AppData\Roaming\Claude\claude_desktop_config.json :: 파일 탐색기에서 직접 열기 :: Win+R → %APPDATA%\Claude 입력
⚠️ Node.js가 없을 때 nodejs.org에서 LTS(Long-Term Support) 버전을 다운로드해서 설치하세요. 설치 후 터미널/명령 프롬프트를 완전히 닫고 다시 열어야 node 명령이 인식됩니다.
🎬 실습 시나리오 01
내 컴퓨터에 Claude Desktop이 MCP 준비가 됐는지 확인
설치를 마친 후, Claude Desktop이 정상적으로 MCP를 인식하는지 확인하는 기본 테스트입니다.
  • Claude Desktop 앱을 열고 새 대화를 시작합니다.
  • 입력창 오른쪽 하단에 🔨 망치 아이콘이 보이면 MCP 준비 완료입니다.
  • 망치 아이콘 클릭 → 연결된 MCP 서버와 사용 가능한 도구 목록 확인.
  • 아직 MCP 서버를 설치하지 않았다면 목록이 비어있을 수 있습니다 (정상).

파일시스템 MCP 연결 — 내 컴퓨터 파일에 접근하기

⏱ 예상 시간 30분  |  핵심 실습

MCP의 가장 기본이자 가장 강력한 기능: 파일시스템 접근. 설정 파일에 몇 줄만 추가하면 클로드가 내 컴퓨터 폴더를 직접 읽고 쓸 수 있습니다. 설정은 단 한 번, 이후로는 자연어로 파일을 다룹니다.

적용 방법 | 파일시스템 MCP 서버 설치 (3단계)
  1. 1
    claude_desktop_config.json 파일 열기

    앞서 확인한 경로의 설정 파일을 텍스트 에디터(메모장, VS Code 등)로 엽니다. 파일이 없으면 빈 파일을 생성합니다.

  2. 2
    아래 JSON 설정 복사해서 붙여넣기

    기존 파일 내용을 모두 지우고, 아래 설정으로 교체합니다. username과 경로를 실제 내 컴퓨터 경로로 수정해야 합니다.

  3. 3
    Claude Desktop 완전 재시작

    파일 저장 후 Claude Desktop을 완전히 종료(트레이에서도 종료) 후 다시 시작합니다. 🔨 아이콘이 나타나면 성공입니다.

파일 경로: ~/Library/Application Support/Claude/claude_desktop_config.json
{ "mcpServers": { "filesystem": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-filesystem", "/Users/여기에_사용자명/Desktop", ← 바탕화면 "/Users/여기에_사용자명/Downloads", ← 다운로드 "/Users/여기에_사용자명/Documents" ← 문서 폴더 ] } } } ━━━ 내 사용자명 확인 방법 ━━━ # 터미널에서 실행: echo $HOME # /Users/johndoe 처럼 나오면 johndoe가 사용자명
파일 경로: C:\Users\[사용자명]\AppData\Roaming\Claude\claude_desktop_config.json
{ "mcpServers": { "filesystem": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-filesystem", "C:\\Users\\여기에_사용자명\\Desktop", "C:\\Users\\여기에_사용자명\\Downloads", "C:\\Users\\여기에_사용자명\\Documents" ] } } } 주의: Windows 경로는 역슬래시를 두 번(\\) 써야 합니다 내 사용자명 확인: Win+R → cmd → echo %USERNAME%
💡 접근 허용 폴더를 신중하게 선택하세요 처음에는 Desktop, Downloads, Documents 정도만 추가하는 것이 안전합니다. 필요에 따라 특정 프로젝트 폴더를 추가할 수 있습니다. 전체 드라이브(C:\\ 또는 /)는 보안상 추가하지 않는 것을 권장합니다.
🎬 실습 시나리오 02
클로드로 내 다운로드 폴더 정리하기
파일시스템 MCP 연결 후 첫 번째 실습. 다운로드 폴더의 파일 목록을 보고, 분류 기준을 제시해달라고 요청합니다. 실제 파일에 접근하는 클로드를 처음으로 경험합니다.
  • Claude Desktop 재시작 후 🔨 아이콘 클릭 → "filesystem" 서버와 도구들 확인
  • 새 대화 시작 → 프롬프트 #1을 입력해 다운로드 폴더 스캔 요청
  • 클로드가 실제 파일 목록을 읽어서 보여주는 것 확인
  • 이어서 프롬프트 #2로 파일 분류 작업 진행

📋 복사해서 바로 쓰는 프롬프트 #1

파일 탐색
MCP 연결 첫 테스트 — 폴더 스캔
내 다운로드 폴더를 살펴봐줘. 아래 내용을 알려줘: 1. 파일이 총 몇 개 있는지 2. 파일 유형별(PDF, 이미지, 문서, 압축파일 등) 분류 3. 가장 최근에 추가된 파일 5개 4. 용량이 큰 파일 상위 3개 (알 수 있다면) 5. 오래된 파일(3개월 이상) 중 정리가 필요해 보이는 것

📋 복사해서 바로 쓰는 프롬프트 #2

파일 내용 분석
특정 파일 내용 읽고 요약하기
내 문서 폴더에서 PDF 파일을 찾아줘. 발견한 PDF 파일들에 대해: 1. 파일명과 대략적인 내용 유형 (계약서, 보고서, 강의자료 등) 2. 가장 최근 파일을 하나 골라 핵심 내용을 3줄로 요약해줘 3. 내가 빠르게 찾을 수 있도록 파일별 1줄 설명을 만들어줘 # 특정 파일만 보고 싶다면: # "문서 폴더의 [파일명.pdf]를 읽고 핵심 내용을 정리해줘"

📋 복사해서 바로 쓰는 프롬프트 #4

파일 검색
자연어로 파일 검색하기
내 문서와 다운로드 폴더에서 "계약" 관련 파일을 모두 찾아줘. 찾은 파일들에 대해: - 파일명과 경로 - 마지막 수정 날짜 - 파일 크기 그리고 가장 최근 계약 관련 파일의 내용을 간단히 요약해줘. 추가로, 2025년에 생성/수정된 파일만 필터링해서 보여줘.

📋 복사해서 바로 쓰는 프롬프트 #5

여러 파일 분석
여러 파일을 한 번에 분석·비교하기 (MCP의 핵심 강점)
내 Documents/reports 폴더에 있는 월간 보고서들을 비교 분석해줘. 분석 내용: 1. 각 보고서의 핵심 지표 수치를 표로 정리 2. 월별 트렌드 변화 (증가/감소/유지) 3. 눈에 띄는 이슈나 특이사항 4. 다음 보고서에 반영해야 할 시사점 ※ 폴더에 파일이 없다면, Documents 폴더에서 비슷한 이름의 파일들을 찾아서 분석해줘.
✅ 파일시스템 MCP 정상 작동 확인 방법 클로드가 "저는 파일에 직접 접근할 수 없습니다"가 아니라 실제 파일 목록을 보여주거나, 파일 내용을 요약해주면 MCP가 정상 작동 중입니다. 입력창 오른쪽 하단의 🔨 아이콘을 클릭하면 현재 사용 가능한 도구 목록을 확인할 수 있습니다.

    설정 파일에 서버를 추가하는 방법

    mcpServers 객체 안에 서버를 이름:설정 형식으로 계속 추가하면 됩니다. 아래 예시는 filesystem + GitHub + SQLite를 동시에 추가합니다.

  2. 2
    API 키가 필요한 서버 처리

    GitHub, Slack 등 API 키가 필요한 서버는 env 필드에 환경 변수로 추가합니다. 토큰 값은 절대 다른 사람과 공유하면 안 됩니다.

  3. 3
    Desktop Extensions 활용 (권장)

    JSON 설정이 어렵다면 Claude Desktop → Settings → Extensions에서 그래픽 인터페이스로 쉽게 추가할 수 있습니다. 점점 더 많은 MCP 서버가 Extensions 형태로 제공되고 있습니다.

(Slack MCP가 연결되어 있다면 #팀채널에 바로 전송해줘도 돼)
보안 가이드 | MCP를 안전하게 사용하는 5가지 원칙
  1. 1
    필요한 폴더만 허용하라

    전체 홈 디렉토리나 시스템 폴더를 추가하지 마세요. 업무에 필요한 특정 폴더만 정확히 지정하세요. 언제든지 설정 파일에서 폴더를 추가/제거할 수 있습니다.

  2. 2
    신뢰할 수 있는 MCP 서버만 설치하라

    공식 Anthropic 서버(@modelcontextprotocol/), Desktop Extensions 디렉토리, 잘 알려진 오픈소스만 사용하세요. 출처 불명의 MCP 서버는 악성 코드를 포함할 수 있습니다.

  3. 3
    API 토큰·비밀번호를 설정 파일에 직접 입력하지 마라

    가능하면 환경 변수를 사용하거나, 최소 권한(Read-only)의 토큰을 사용하세요. 설정 파일을 Git에 올리지 마세요.

  4. 4
    파일 삭제는 항상 확인하라

    클로드에게 파일 삭제를 요청할 때는 "삭제 전에 반드시 목록을 보여주고 확인을 받아라"고 명시하세요. 실수로 중요한 파일이 삭제될 수 있습니다.

  5. 5
    민감 정보가 있는 폴더는 제외하라

    금융 정보, 개인 식별 정보, 비밀번호 관련 파일이 있는 폴더는 MCP 접근 경로에서 제외하세요.

문제 해결 / 디버깅
MCP가 작동하지 않을 때 — 트러블슈팅 체크리스트
━━━ 문제 현상별 해결법 ━━━ 문제 1: 🔨 아이콘이 안 보임 → Claude Desktop 완전 종료 (트레이 아이콘 우클릭 → 종료) 후 재시작 → 설정 파일 JSON 문법 오류 확인 (jsonlint.com에서 검사) 문제 2: 아이콘은 있는데 파일 접근 안 됨 → 설정 파일의 경로가 실제 존재하는지 확인 → macOS: 경로에 한글이나 특수문자 없는지 확인 → Node.js가 최신 버전인지 확인 (node --version) 문제 3: "Permission denied" 오류 → macOS: 시스템 환경설정 → 개인 정보 보호 → 전체 디스크 접근 → Claude Desktop에 권한 부여 → Windows: 관리자 권한으로 Claude Desktop 실행 문제 4: 로그 확인 방법 macOS 터미널: tail -f ~/Library/Logs/Claude/mcp*.log Windows 명령 프롬프트: type "%APPDATA%\Claude\logs\mcp*.log" 위 방법으로도 해결 안 된다면 로그 내용을 클로드에게 보여주면서 "이 MCP 오류를 해결하는 방법을 알려줘"라고 질문하세요.

📌 20일차 핵심 정리

MCP란?

클로드와 외부 데이터·도구를 연결하는 USB-C 같은 오픈 표준. 파일, DB, API 등에 실시간 접근 가능.

필수 준비물

Claude Desktop 앱 + Node.js. claude_desktop_config.json에 서버 설정 추가 후 재시작.

파일시스템 MCP

@modelcontextprotocol/server-filesystem 패키지. 허용 폴더를 지정하면 클로드가 파일을 직접 읽기·쓰기·검색.

MCP vs 스킬

스킬 = 클로드에게 방법을 가르침. MCP = 클로드에게 새 능력 부여. 둘을 함께 쓰면 최강.

보안 핵심

필요한 폴더만 허용 + 신뢰된 서버만 + 민감 폴더 제외 + 파일 삭제 시 항상 확인 요청.
