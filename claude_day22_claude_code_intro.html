📖 핵심 개념

Claude Code란? — "코드베이스를 이해하는 AI"

지금까지 AI 코딩 도구는 파일 하나, 함수 하나를 보조하는 수준이었습니다. Claude Code는 다릅니다. 프로젝트 전체를 읽고, 파일 간 의존성을 파악하고, Git 히스토리를 이해하며, 터미널 명령을 직접 실행합니다.

Claude Code는 터미널(CLI)에서 실행되는 에이전트형 AI 코딩 도구입니다. "이 기능을 추가해줘"라고 말하면 관련 파일을 찾아 코드를 수정하고, 테스트를 실행하고, Git 커밋까지 수행합니다. 단순한 자동완성이 아니라 코드를 이해하고 실행하는 AI 동료입니다.

❌ 기존 AI 코딩 도구

파일 하나를 복사해서 붙여넣고 → AI가 수정안 제시 → 수동으로 파일에 적용. 프로젝트 전체 맥락은 모름. 매번 컨텍스트 재설명 필요.

✅ Claude Code

프로젝트 폴더에서 실행 → 전체 코드베이스 자동 파악 → 자연어로 지시 → 여러 파일 동시 수정 + 테스트 실행 + Git 커밋까지 자동.

Claude Code — 할 수 있는 것들
# Claude Code가 할 수 있는 것 ✓ 파일 읽기·쓰기·생성·삭제 → 여러 파일 동시 수정 가능 ✓ 터미널 명령 실행 → npm install, pytest, git 등 ✓ Git 작업 → commit, branch, diff 분석 ✓ 코드베이스 전체 탐색 → 1M 토큰 컨텍스트 윈도우 ✓ 버그 찾기 & 수정 → 에러 로그 분석 후 자동 수정 ✓ 테스트 작성 & 실행 → TDD 워크플로우 지원 ✓ 리팩토링 → 전체 프로젝트 패턴 일관성 유지 ✓ 문서 자동 작성 → README, API 문서, 주석

L1

설치 & 인증 — 첫 번째 $ claude 실행까지

⏱ 예상 시간 20분  |  한 번만 하면 끝

Claude Code는 2026년부터 네이티브 바이너리 설치 방식이 공식 권장됩니다. Node.js 없이 설치 가능하고, 자동 업데이트가 지원됩니다. npm을 통한 설치도 여전히 가능하지만 공식적으로는 Deprecated(더 이상 권장 안 함) 상태입니다.

적용 방법 | 설치 3단계
  1. 1
    터미널 열기

    macOS: Cmd+Space → "Terminal" 또는 "iTerm" 검색. Windows: WSL 설치 후 Ubuntu 앱 실행. Linux: 기본 터미널 앱 사용.

  2. 2
    설치 명령 실행

    아래 OS별 명령어를 복사해 터미널에 붙여넣고 Enter. 자동으로 다운로드·설치·PATH 등록이 완료됩니다.

  3. 3
    새 터미널 창 열고 인증

    설치 후 새 터미널 창을 열고 claude를 입력합니다. 브라우저가 자동으로 열리며 Claude.ai 계정으로 로그인하면 인증 완료입니다.

Terminal — macOS/Linux 설치
# ── 1단계: 네이티브 바이너리 설치 (권장) ────────────── $ curl -fsSL https://claude.ai/install.sh | bash # 설치 완료 후 새 터미널 창 열기 (중요!) # PATH가 새 창에서만 인식됨 # ── 2단계: 설치 확인 ──────────────────────────────── $ claude --version # Claude Code 1.x.x 처럼 버전이 나오면 성공 # ── 3단계: 진단 도구로 환경 확인 ─────────────────── $ claude doctor # 설정 문제를 자동 감지하고 수정 방법 안내 # ── 4단계: 첫 실행 & 인증 ────────────────────────── $ claude # 브라우저가 자동으로 열림 → Claude.ai 계정으로 로그인 # 인증 후 터미널로 돌아오면 세션 시작
PowerShell — Windows 설치
# ── Windows: PowerShell (관리자 권한 불필요) ────────── # Win+X → "Windows PowerShell" 실행 # 방법 1: 네이티브 설치 스크립트 (권장) PS> irm https://claude.ai/install.ps1 | iex # 또는 winget으로 설치 PS> winget install Anthropic.ClaudeCode # ── WSL(Ubuntu) 에서 설치하는 방법 ────────────────── # WSL이 없다면: PowerShell에서 먼저 실행 PS> wsl --install # 재시작 후 Ubuntu 앱 열기 # Ubuntu 터미널에서: $ curl -fsSL https://claude.ai/install.sh | bash # 설치 확인 $ claude --version $ claude doctor
npm 방식 — Node.js 18+ 필요
# ⚠️ npm 방식은 Deprecated. 네이티브 설치를 권장합니다. # 특정 버전 고정이 필요한 경우에만 사용하세요. # Node.js 18+ 설치 확인 $ node --version # v18.0.0 이상이 필요 # npm으로 전역 설치 (⚠️ sudo 절대 사용 금지) $ npm install -g @anthropic-ai/claude-code # 설치 확인 $ claude --version # EACCES 오류가 나면 (sudo 쓰지 말고 nvm 사용): $ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash $ source ~/.bashrc $ nvm install --lts $ npm install -g @anthropic-ai/claude-code # 나중에 네이티브로 전환하려면: $ curl -fsSL https://claude.ai/install.sh | bash $ npm uninstall -g @anthropic-ai/claude-code
🎬 실습 시나리오 01
설치 완료 후 — 첫 번째 claude 실행 경험하기
설치가 끝났다면 실제 프로젝트 폴더가 없어도 첫 실행을 경험할 수 있습니다. 간단한 폴더를 만들고 Claude Code의 기본 인터페이스를 탐색해봅니다.
  • 새 터미널 창을 열고 claude --version으로 설치 확인
  • 테스트용 폴더 생성: mkdir my-first-cc && cd my-first-cc
  • claude 입력 → 브라우저 인증 완료
  • 터미널에 나타난 Claude Code 인터페이스에서 /help 입력해 명령어 목록 확인
  • /exit 또는 Ctrl+C로 종료

기본 사용법 — 슬래시 커맨드와 Plan Mode

⏱ 예상 시간 35분  |  핵심 조작법 마스터

Claude Code 안에서 대화는 두 가지 방식으로 합니다. 첫째, 자연어로 작업을 지시하는 일반 대화. 둘째, 슬래시 커맨드(/)로 세션을 제어하는 명령어 방식. 이 둘을 자유롭게 섞어 쓰는 것이 핵심입니다.

가장 많이 쓰는 슬래시 커맨드

커맨드역할언제 사용?
/help사용 가능한 모든 명령어 목록 표시처음 시작할 때, 기억 안 날 때
/planPlan Mode 토글 — 실행 전 계획 먼저 확인복잡한 작업 전 반드시 사용
/clear대화 히스토리 초기화 (파일 수정은 유지)새 작업 시작할 때
/compact대화 내용을 요약으로 압축 (토큰 절약)긴 세션이 느려질 때
/undo직전 파일 수정 되돌리기변경이 마음에 안 들 때
/model사용할 Claude 모델 변경Sonnet ↔ Opus 전환
/exitClaude Code 세션 종료작업 완료 후

CLAUDE.md — 프로젝트 메모리 설정하기

⏱ 예상 시간 30분  |  Claude Code의 핵심 비밀 병기

Claude Code를 시작할 때마다 매번 "우리 팀은 TypeScript를 쓰고, 코딩 스타일은 이렇고, 테스트는 Jest로 해"를 설명해야 한다면 끔찍할 겁니다. CLAUDE.md는 이것을 해결하는 프로젝트 메모리 파일입니다. 프로젝트 루트에 놓아두면 Claude Code가 세션 시작 때마다 자동으로 읽고 기억합니다.

적용 방법 | CLAUDE.md 작성 원칙
  1. 1
    두 종류의 CLAUDE.md 이해하기

    글로벌 (~/.claude/CLAUDE.md): 모든 프로젝트에 적용되는 개인 설정. 코딩 스타일, 선호하는 언어, 자주 쓰는 패턴 등.
    프로젝트 (./CLAUDE.md): 해당 프로젝트에만 적용. 기술 스택, 팀 컨벤션, 실행 방법 등.

  2. 2
    CLAUDE.md에 넣을 것과 넣지 말 것

Claude Code란?

터미널 기반 에이전트형 AI. 전체 코드베이스를 읽고, 파일을 직접 수정하고, 명령을 실행하는 진짜 AI 동료.

설치 방법

네이티브 설치 (권장): curl -fsSL https://claude.ai/install.sh | bash. Node.js 불필요. 자동 업데이트.

핵심 커맨드

/plan (계획 먼저), /clear (초기화), /compact (토큰 절약), /undo (되돌리기). 나머지는 자연어로 충분.

Plan Mode

3개 이상 파일 수정 시 필수. 실행 전 계획 확인 → 승인 → 정확한 실행. 37% 높은 성공률.

CLAUDE.md

프로젝트 메모리 파일. 기술 스택, 컨벤션, 주의사항 작성. 매 세션 자동 로드. 글로벌 + 프로젝트별 2종.
