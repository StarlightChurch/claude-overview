📖 핵심 개념 & 원리

Computer Use란? — "AI가 인간처럼 컴퓨터를 다룬다"

2024년 10월, Anthropic이 발표한 Computer Use는 AI 역사에서 전환점이 되는 기능입니다. 클로드가 단순히 텍스트를 생성하는 것을 넘어, 화면을 보고 → 상황을 판단하고 → 마우스를 클릭하고 → 키보드를 타이핑하는 행동을 직접 수행합니다.

마치 처음 컴퓨터 앞에 앉은 사람처럼, 클로드는 화면을 스캔하고 어디를 눌러야 할지 파악합니다. 인간이 하는 방식과 동일하게 — 하지만 24시간, 실수 없이.

안전한 사용을 위한 핵심 원칙

✓ 해야 할 것
  • 샌드박스(가상 환경)에서 먼저 테스트
  • 중요 파일 사전 백업
  • 작업 범위를 명확히 제한해서 지시
  • 실행 중 화면 모니터링
  • 민감 정보(비번·카드번호) 입력 지양
  • 필요한 권한만 최소로 부여
✗ 피해야 할 것
  • 무인 상태로 장시간 실행
  • 시스템 파일·설정에 접근 허용
  • 금융 거래·결제 자동화 (검토 없이)
  • 여러 앱 동시 제어 요청
  • 목적이 불명확한 광범위한 지시
  • 프롬프트 인젝션 가능성 있는 환경
💡 "제한된 자율성(Bounded Autonomy)" 원칙 Anthropic이 설계한 Computer Use의 핵심 안전 철학입니다. 클로드는 사용자가 승인한 범위 안에서만 자율적으로 작동합니다. 그 범위를 벗어나는 작업이 필요하면 반드시 추가 승인을 요청합니다. 명확한 경계를 설정할수록 더 안전하고 예측 가능한 자동화가 가능합니다.
L1

Claude Cowork — 비개발자를 위한 컴퓨터 제어

⏱ 예상 시간 25분  |  코딩 불필요 · GUI 기반

Claude Cowork는 2026년 1월 출시된 비개발자용 데스크탑 에이전트입니다. 터미널 없이 GUI 인터페이스에서 폴더를 지정하고 자연어로 지시하면, 클로드가 파일 시스템과 브라우저, 앱을 직접 조작합니다. Computer Use 기술의 가장 접근하기 쉬운 형태입니다.

적용 방법 | Claude Cowork에서 컴퓨터 제어 활용
  1. 1
    Cowork 실행 및 작업 폴더 지정 Claude Desktop 앱 → Cowork 탭 실행. "세션 시작" 클릭 후 작업할 로컬 폴더를 선택합니다. 이 폴더가 클로드가 접근할 수 있는 범위가 됩니다.
  2. 2
    작업 목표를 결과 중심으로 설명 "어떻게 해줘"가 아니라 "이런 결과가 필요해" 방식으로 지시합니다. 클로드가 화면을 보면서 스스로 방법을 찾습니다. "~/Downloads 폴더의 PDF 파일들을 날짜별로 정리해줘"처럼요.
  3. 3
    진행 상황 모니터링 및 개입 Cowork는 작업 로그를 실시간으로 보여줍니다. 예상과 다른 방향으로 진행되면 언제든 "잠깐, 이 파일은 건드리지 마"처럼 즉시 개입할 수 있습니다. 완전히 눈을 떼지 마세요.
  4. 4
    Chrome 확장으로 웹 자동화 추가 Claude Chrome Extension을 설치하면 브라우저 조작도 가능해집니다. "네이버에서 검색하고 결과를 정리해줘" 같은 웹 포함 워크플로우를 실행할 수 있습니다.
🎬 실습 시나리오 01
다운로드 폴더 영수증 사진을 자동으로 분류·정리하기
스마트폰으로 찍어 저장된 영수증 사진들이 Downloads 폴더에 뒤섞여 있습니다. Cowork에게 자동 분류를 맡겨봅니다. 이것이 Computer Use의 가장 기초적이고 실용적인 첫 경험입니다.
  • Claude Desktop → Cowork 탭 → 세션 시작 → Downloads 폴더 선택
  • 아래 프롬프트 #1을 입력하고 실행 → 클로드가 파일을 분석하는 것을 지켜봄
  • 진행 로그 확인 — "어떤 파일을 어디로 옮기려고 하는지" 확인
  • 결과 검토 후 만족스러우면 "계속 진행해줘", 수정 필요하면 즉시 피드백

📋 복사해서 바로 쓰는 프롬프트 #1

Cowork — 파일 정리
Claude Cowork 입력창에 붙여넣기
Downloads 폴더를 분석해서 파일을 자동 정리해줘. == 정리 규칙 == 사진/이미지 파일 (.jpg, .png, .heic): → 파일명에 날짜가 있으면: 연도-월 폴더로 (예: 2025-06/) → 날짜 없으면: "사진_미분류/" 폴더로 문서 파일 (.pdf, .docx, .xlsx): → 파일명에 "영수증" 또는 숫자만 있으면: "영수증/" 폴더로 → 계약, 견적, 제안: "계약서류/" 폴더로 → 나머지: "문서_기타/" 폴더로 압축파일 (.zip, .rar): → "압축파일/" 폴더로 == 주의사항 == - 파일을 삭제하지 말고 이동만 해줘 - 이동 전에 반드시 어떤 파일을 어디로 옮길지 목록으로 보여줘 - 내가 "진행해줘"라고 하면 그때 실행해줘 - 최근 3일 이내 파일은 건드리지 마 (혹시 사용 중일 수 있으니)

📋 복사해서 바로 쓰는 프롬프트 #2

Cowork — 문서 일괄 처리
여러 문서를 한 번에 처리하기
Documents/monthly-reports 폴더에 있는 월간 보고서 엑셀 파일들을 분석하고 요약해줘. == 처리 방법 == 1. 폴더 내 모든 .xlsx 파일 목록 확인 2. 각 파일을 열어서 주요 지표(매출, 비용, 이익) 읽기 3. 아래 형식으로 통합 요약 파일 작성: 파일명: 2025년_월별_성과_요약.xlsx 내용: | 월 | 매출 | 비용 | 순이익 | 전월대비 | |---|-----|-----|------|--------| 4. 요약 파일을 같은 폴더에 저장 엑셀 파일 형식이 제각각일 수 있어. 읽기 어려운 파일은 건너뛰고 "[파일명] — 형식 불일치"로 표시해줘. 처리 전 파일 목록 먼저 보여주고 확인 후 진행해.

Cowork — 브라우저 자동화 (Chrome 확장 필요)
Chrome Extension 연결 후 — 웹 + 파일 통합 작업
아래 작업을 순서대로 수행해줘. 1단계: 웹에서 정보 수집 네이버 뉴스에서 오늘 "AI 스타트업" 관련 기사 5개를 찾아줘. 각 기사의 제목, URL, 요약 1줄을 수집해줘. 2단계: 파일에 저장 수집한 정보를 Desktop/뉴스클리핑/ 폴더에 오늘 날짜로 마크다운 파일로 저장해줘. (파일명: YYYY-MM-DD_AI스타트업_뉴스.md) 파일 형식: # AI 스타트업 뉴스 클리핑 — [날짜] ## [기사 제목] - 출처: 네이버 뉴스 - URL: [링크] - 요약: [1줄] --- 폴더가 없으면 만들어줘. 기사가 5개 미만이면 최대한 수집하고 "X개 수집 완료"로 보고해줘.

Claude Code Computer Use — 개발자 데스크탑 자동화

⏱ 예상 시간 30분  |  Pro/Max · macOS 연구 프리뷰

2026년 3월 30일, Anthropic이 Claude Code에 Computer Use를 통합했습니다. 이제 Claude Code는 코드를 작성하는 것을 넘어, 실제로 앱을 열고, UI를 클릭하고, 빌드한 것을 직접 테스트합니다. 개발자의 완전한 자동화 루프가 완성됩니다.

⚠️ 현재 상태 (2026년 4월 기준) Claude Code Computer Use는 macOS 연구 프리뷰 상태입니다. Claude Code v2.1.85 이상이 필요하며, macOS 접근성(Accessibility)과 화면 녹화(Screen Recording) 권한을 허용해야 합니다. Windows/Linux는 아직 지원되지 않습니다.
적용 방법 | Claude Code에서 Computer Use 활성화
  1. 1
    macOS 시스템 권한 부여 시스템 설정 → 개인 정보 보호 및 보안 → 접근성과 화면 녹화에서 Claude Code(터미널)를 허용합니다. 이 권한 없이는 화면 캡처와 UI 조작이 불가능합니다.
  2. 2
    Claude Code 세션에서 /mcp 명령으로 활성화 프로젝트 폴더에서 claude 실행 → /mcp 입력 → computer-use 서버 선택. 또는 settings.json에 MCP 서버로 등록하면 자동 활성화됩니다.
  3. 3
    Claude Code Desktop의 미리보기 패널 활용 Claude Code Desktop 앱에서는 내장 브라우저 미리보기 패널을 통해 빌드한 앱을 직접 테스트합니다. "Preview" 탭에서 클로드가 스스로 UI를 클릭하며 검증합니다.
Terminal — Claude Code Computer Use 설정
# ── 1. 버전 확인 (v2.1.85+ 필요) ───────────────── $ claude --version # Claude Code 2.1.xx 이상이어야 합니다 # ── 2. macOS 권한 확인 ──────────────────────────── # 시스템 설정 → 개인 정보 보호 → 접근성에서 # "터미널" 또는 실행 중인 터미널 앱에 체크 # 화면 녹화도 동일하게 허용 # ── 3. Claude Code 실행 ────────────────────────── $ cd my-project $ claude # ── 4. MCP로 computer-use 활성화 ───────────────── > /mcp # 목록에서 computer-use 선택 # ── 또는 settings.json에 영구 등록 ─────────────── # ~/.claude/settings.json에 추가: { "mcpServers": { "computer-use": { "command": "claude", "args": ["mcp", "serve", "computer-use"] } } }
🎬 실습 시나리오 02
Claude Code가 빌드 → 앱 열기 → 버그 찾기 → 수정까지 자동 완성
전통적으로 개발자가 코드 작성 → 직접 실행 → UI 확인 → 버그 메모 → 코드 수정을 반복해야 했습니다. Claude Code Computer Use로 이 전체 루프가 자동화됩니다.
  • 프로젝트 폴더에서 claude 실행 → /mcp로 computer-use 활성화
  • 프롬프트 #4 입력: "이 기능을 실제로 열어서 테스트해줘"
  • 클로드가 dev 서버 시작 → 브라우저 열기 → UI 클릭 테스트를 자동 수행하는 것 관찰
  • 발견된 버그를 자동으로 수정하는지 확인

📋 복사해서 바로 쓰는 프롬프트 #4

API 기반 Computer Use — 커스텀 자동화 시스템 구축

⏱ 예상 시간 30분  |  개발자 전용 · 완전한 제어

API를 통한 Computer Use는 가장 강력하고 유연합니다. Docker 컨테이너 안에 가상 데스크탑을 구성하고, Python 코드로 클로드에게 GUI 작업을 지시합니다. 기업 내부 시스템 자동화, 테스트 파이프라인 구축에 활용됩니다.

💡 왜 Docker 컨테이너에서 실행하는가? API 기반 Computer Use는 실제 데스크탑이 아니라 격리된 가상 환경(Docker)에서 실행하는 것이 안전합니다. Anthropic의 공식 레퍼런스 구현(claude-quickstarts)을 사용하면 격리된 환경, 웹 UI, 도구 구현이 모두 준비된 상태로 시작할 수 있습니다.
적용 방법 | API Computer Use 아키텍처 이해
  1. 1
    3개 도구가 협력하는 구조 computer 도구: 화면 캡처 + 마우스/키보드 제어. bash 도구: 터미널 명령 실행. text_editor 도구: 파일 직접 편집. 이 셋을 함께 사용하면 GUI 조작 + 파일 처리 + 명령 실행을 통합한 완전한 자동화가 가능합니다.
  2. 2
    에이전트 루프 패턴 이해 단순 API 호출이 아닙니다. 클로드의 응답에 "tool_use" 블록이 오면 해당 액션을 실행하고, 결과를 다시 API로 전송하는 루프를 완료까지 반복합니다. 이 루프를 직접 구현하거나 공식 레퍼런스 코드를 활용합니다.
  3. 3
    프롬프트에서 안전 제약 명시 API 호출 시 시스템 프롬프트에 "이 환경은 테스트 환경입니다. 외부 서비스에 실제 요청을 보내지 마세요"처럼 명확한 제약을 포함해야 합니다. 특히 웹 접근이 가능한 환경에서는 더욱 중요합니다.

"이 페이지에서만 작업. 다른 탭이나 사이트로 이동하지 마." → 명확한 경계가 안전한 자동화의 핵심 ━━━ 패턴 5: 확인 요청 ━━━ ✅ "삭제, 전송, 결제 등 되돌릴 수 없는 작업 전에는 반드시 확인 요청." → 되돌릴 수 없는 액션은 항상 인간 검토
현실적인 한계 — 알고 쓰면 더 잘 쓴다
  1. 1
    속도 — API 왕복 시간 각 화면 캡처마다 API 호출이 발생합니다. 사람이 직접 하는 것보다 3~10배 느릴 수 있습니다. 빠른 반복 작업보다 복잡하지만 속도가 중요하지 않은 작업에 적합합니다.
  2. 2
    좌표 정확도 — 작은 UI 요소 매우 작거나 겹쳐있는 버튼을 정확히 클릭하기 어려울 수 있습니다. 해결: 화면 해상도를 높이거나, 줌 기능 사용, 또는 요소를 텍스트로 명확히 지시합니다.
  3. 3
    동적 UI — 로딩·애니메이션 로딩 중인 화면이나 애니메이션 중인 UI를 오인할 수 있습니다. 중요한 단계에는 "X초 기다린 후 진행" 또는 "로딩 완료를 확인 후 진행"을 명시합니다.
  4. 4
    비용 — 이미지 처리 토큰 스크린샷마다 이미지 처리 비용이 발생합니다. 긴 자동화 세션은 상당한 토큰을 소비합니다. 개발/테스트 환경에서 먼저 검증하고 프로덕션에 적용하세요.

📌 25일차 핵심 정리

Computer Use란?

클로드가 스크린샷으로 화면을 보고, 마우스/키보드로 직접 조작하는 기능. 2024년 10월 출시. RPA보다 유연하고 자연어로 지시 가능.

비전-액션 루프

스크린샷 → 분석·판단 → 액션 실행 → 결과 확인의 4단계가 완료까지 반복. 이 루프가 Computer Use의 핵심 원리.

3가지 접근법

Cowork(비개발자·GUI), Claude Code Desktop(개발자·터미널), API+Docker(완전 제어·기업용). 목적에 맞게 선택.

안전 원칙

샌드박스 먼저, 범위 명시, 되돌릴 수 없는 액션은 승인 요청, 중요 파일 백업. "제한된 자율성" 원칙 준수.

효과적인 프롬프트

방법보다 목표 중심, 실패 처리 명시, 단계별 체크포인트, 접근 범위 제한, 되돌릴 수 없는 작업 승인 요청.
