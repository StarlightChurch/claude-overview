2025년 9월 이후 바뀐 Claude의 데이터 정책

2025년 9월 28일, Anthropic은 소비자용 클로드의 데이터 정책을 크게 변경했습니다. 이전까지는 "사용자 데이터를 학습에 절대 사용하지 않음"이 기본이었지만, 이제 기본값이 바뀌었습니다. 모른 채 넘어갔다면 지금 대화가 AI 학습에 사용되고 있을 수 있습니다.

📋 2025년 9월 이후 현행 데이터 정책 — 플랜별 요약
⚠️

Free / Pro / Max 플랜 — 기본값: 학습 사용 (opt-out 가능)

기본적으로 대화 데이터가 모델 학습에 사용됩니다. 설정에서 직접 끄지 않으면 새 대화가 학습 데이터로 활용되며, 보존 기간이 최대 5년으로 늘어납니다. 끄면 30일 보존 후 삭제됩니다.

✅

Team / Enterprise 플랜 — 기본값: 학습 미사용 (보장)

Claude for Work(Team, Enterprise)는 이 정책 변경에서 제외됩니다. 기업 계약(Commercial Terms) 하에 운영되며, 데이터는 절대 모델 학습에 사용되지 않습니다.

✅

API 사용 (개발자) — 학습 미사용 (기본 보장)

Anthropic API를 직접 사용하는 경우(Amazon Bedrock, Google Vertex AI 포함), 데이터는 모델 학습에 사용되지 않습니다. API 사용 시 7일 보존 후 삭제가 기본입니다.

🔒

시크릿(Incognito) 대화 — 항상 학습 제외

모든 플랜에서 시크릿 대화(Incognito Chat)는 설정과 무관하게 학습에 사용되지 않습니다. 민감한 내용을 다룰 때 활용하세요.

👍

좋아요/싫어요 피드백 — 5년 보존

응답에 thumbs up/down을 누르면 해당 전체 대화가 5년 동안 보존됩니다. 민감한 대화에서 피드백 버튼을 누르지 않는 것이 좋습니다.

⚠️ 설정 확인이 시급합니다 Free/Pro/Max 플랜 사용자라면 지금 바로 설정을 확인하세요. claude.ai → 설정 → 개인 정보 보호에서 "모델 개선을 위해 데이터 사용" 옵션이 켜져 있으면 새 대화가 학습에 활용됩니다. 아래 L1에서 상세 방법을 안내합니다.

설정 관리 — 지금 당장 해야 할 5가지

⏱ 예상 시간 20분  |  모든 플랜 해당

개념을 이해했다면 지금 바로 설정을 점검해야 합니다. 5분이면 완료되는 기본 보안 설정을 하나씩 처리합니다.

적용 방법 | Claude.ai 보안 설정 체크리스트
  1. 1
    모델 학습 데이터 사용 설정 끄기 (최우선)

    아래 경로로 접근해서 "모델 개선" 옵션을 OFF로 변경합니다.

Claude.ai 접속 → 우측 상단 프로필 → Settings(설정) → Privacy → "Help improve Claude" 토글 OFF
  1. 2
    대화 기록 저장 설정 확인

    Settings → Privacy에서 "Chat History" 설정을 확인합니다. 대화 기록 저장이 켜져 있으면 이전 대화들이 서버에 보관됩니다. 민감한 업무라면 필요한 대화만 남기고 정기적으로 삭제하세요.

  2. 3
    과거 대화 일괄 삭제

    Settings → Privacy → "Delete all conversations"로 기존 대화를 모두 삭제할 수 있습니다. 단, 삭제 후에도 Anthropic 서버에서 30일 이내 최종 삭제됩니다. 이미 학습에 사용된 데이터는 제거되지 않습니다.

  3. 4
    시크릿 대화(Incognito) 활용법 익히기

    민감한 업무 대화는 시크릿 모드를 사용합니다. Claude.ai 좌측 메뉴에서 "New Incognito Chat" 또는 대화창 우측 메뉴에서 찾을 수 있습니다. 세션이 끝나면 서버에 저장되지 않습니다.

  4. 5
    Claude for Chrome 권한 검토

    Chrome 확장을 설치했다면 브라우저 방문 페이지 데이터도 처리됩니다. chrome://extensions에서 Claude 확장의 권한을 확인하고, 필요한 최소한의 권한만 허용하세요.

⏱ 예상 시간 25분  |  실무 핵심

학습 설정을 끄더라도 대화는 일정 기간 서버에 저장됩니다. 그리고 가끔은 직원이 서비스 품질 개선 목적으로 볼 수 있습니다. 무엇을 AI에게 말해도 되는지를 아는 것이 진짜 보안의 시작입니다.

입력 정보 위험도 분류

🔴 절대 입력 금지
  • 주민등록번호, 여권번호
  • 신용카드 번호·CVV
  • 비밀번호, API 키, 접속 토큰
  • 환자 의료 기록 (개인 식별 포함)
  • 거래처 기밀 계약 원본
  • 미공개 재무 정보
  • 직원 개인정보 원본
🟡 익명화 후 사용
  • 고객 데이터 → 이름·연락처 제거
  • 직원 이름 → 역할명으로 대체
  • 실제 회사명 → 가명 처리
  • 내부 프로젝트명 → 코드명으로
  • 실제 금액 → 비율/% 로 변환
  • 거래처 특정 정보 → 일반화
🟢 안전하게 입력 가능
  • 공개된 일반 정보
  • 익명화된 데이터·시나리오
  • 코드 (민감 정보 제거 후)
  • 구조와 형식만 있는 샘플 데이터
  • 공개 자료 분석 요청
  • 일반적인 업무 문서 초안
적용 방법 | 민감 정보를 안전하게 처리하는 익명화 기법
  1. 1
    이름 → 역할명/코드명으로 대체 "김철수 대표님이 말씀하셨는데" → "A사 CEO가 언급했는데"처럼 실명 대신 역할 또는 익명 코드를 사용합니다. 분석 목적이라면 실명이 필요 없는 경우가 대부분입니다.
  2. 2
    수치 → 상대값/비율로 변환 "3분기 매출 148억 원"을 "3분기 매출 X원(전분기 대비 12% 증가)"처럼 실제 절대값 대신 상대적 수치나 비율로 변환해서 입력합니다.
  3. 3
    코드의 민감 정보 제거 후 입력 코드에 포함된 API 키, DB 접속 정보, 하드코딩된 비밀번호 등을 YOUR_API_KEY, DB_PASSWORD처럼 플레이스홀더로 교체한 후 입력합니다.
  4. 4
    시크릿 모드 + 익명화 = 이중 보호 민감 업무는 시크릿 대화를 시작하고, 그 안에서도 개인식별 정보를 제거한 익명화 데이터를 사용하면 이중으로 보호됩니다.

팀·기업을 위한 AI 보안 가이드라인 구축

⏱ 예상 시간 25분  |  관리자·팀리더 필수

팀원들이 각자 임의로 AI를 쓰고 있다면, 어디선가 기밀 정보가 AI에게 전달되고 있을 수 있습니다. 팀 차원의 AI 사용 가이드라인을 만들고 공유하는 것이 가장 효과적인 예방책입니다.

✅ Team/Enterprise 플랜의 보안 이점 Team, Enterprise 플랜 사용자는 이미 데이터 학습 걱정이 없습니다. 추가로 Enterprise는 SSO(단일 로그인), 감사 로그, 도메인별 사용 제한, 커스텀 데이터 보존 정책을 제공합니다. 민감 데이터를 다루는 팀이라면 Enterprise 플랜 전환을 적극 고려하세요.
적용 방법 | 팀 AI 사용 가이드라인 5요소
  1. 1
    입력 금지 항목 명확히 정의 "고객 개인정보, 미공개 재무 데이터, 보안 자격증명, 법적 계약 원본"처럼 팀 환경에 맞는 입력 금지 항목을 구체적으로 문서화합니다. 애매한 경우도 포함해서 예시를 추가하세요.
  2. 2
    승인된 도구 목록 관리 팀이 사용 가능한 AI 도구를 명시적으로 승인합니다. Claude 웹, Claude Code, 특정 플러그인 등. 승인되지 않은 도구 사용을 금지하고, 새 도구 도입 시 검토 프로세스를 만듭니다.
  3. 3
    AI 생성 결과물 검토 의무화 AI가 작성한 보고서, 코드, 계약서는 반드시 사람이 최종 검토합니다. 특히 법적 효력이 있는 문서나 중요 결정에는 AI 생성 사실을 명시하고 담당자 승인을 받도록 합니다.
  4. 4
    AI 활용 내용 기록 및 감사 중요한 업무에 AI를 사용한 경우 "어떤 프롬프트로, 어떤 목적으로 사용했는지"를 간단히 기록합니다. 나중에 결과물의 출처를 확인하거나 보안 사고 발생 시 추적에 필요합니다.
  5. 5
    정기 교육 및 정책 업데이트 AI 도구와 정책은 빠르게 변합니다. 분기별로 팀 보안 가이드라인을 검토하고 업데이트합니다. 새 팀원 온보딩 시 AI 사용 정책 교육을 필수로 포함합니다.
🎬 실습 시나리오 03
우리 팀을 위한 AI 보안 사용 가이드 문서 자동 생성
클로드를 사용해서 우리 팀에 맞는 AI 보안 사용 가이드라인 문서를 직접 만들어봅니다. 이 문서를 팀 노션, 슬랙, 이메일로 공유하세요.
  • 프롬프트 #5로 팀 맞춤형 AI 사용 가이드 문서 초안 생성
  • 우리 팀의 특수한 상황(업종, 다루는 데이터 유형)을 반영해 수정
  • 법무/보안 담당자가 있다면 검토 후 최종본 배포
  • 팀 공유 폴더나 노션에 저장 + 월례 검토 일정 잡기

팀 AI 보안 가이드라인 문서 생성
팀 리더·관리자가 활용하는 가이드라인 생성기
우리 팀을 위한 AI 사용 보안 가이드라인 문서를 작성해줘. == 팀 정보 == 업종/부서: [예: IT 스타트업 마케팅팀 / 법무팀 / 의료기관] 팀 규모: [예: 8명] 주요 업무: [예: 콘텐츠 제작, 고객 분석, 계약서 검토] 사용 중인 AI 도구: [예: Claude Pro, ChatGPT, GitHub Copilot] 가장 우려되는 점: [예: 고객 개인정보 유출, 기밀 코드 유출] == 문서 구성 == 1. 한 줄 요약 (경영진용) "우리 팀이 AI를 사용할 때 지켜야 할 핵심 원칙은?" 2. 입력 금지 항목 우리 업종에 맞는 구체적인 예시와 함께 3. 사용 가능 항목 안전하게 활용 가능한 업무 유형 예시 4. 상황별 행동 지침 "이런 경우엔 이렇게 해줘" 형식으로 3~5가지 시나리오 5. 의심 상황 보고 절차 "혹시 민감 정보가 유출됐다면 어떻게?" 6. 참고 링크 Anthropic 개인정보처리방침, 팀 담당자 연락처 자리 == 형식 == - 팀원들이 바로 읽고 이해할 수 있는 쉬운 언어 - 법적 효력보다 실용성 중심 - 분량: A4 2페이지 이내 - 마크다운 형식 (노션에 바로 붙여넣기 가능)

프롬프트 인젝션 방어 — AI 보안 취약점 대응
AI를 통한 보안 위협을 이해하고 대응하기
AI 사용 시 발생할 수 있는 보안 위협과 대응 방법을 알려줘. 특히 프롬프트 인젝션이 뭔지. 설명해줄 내용: 1. 프롬프트 인젝션이란 무엇이고 어떻게 발생하는가 2. 내가 AI에게 외부 콘텐츠(이메일, 웹 링크, 첨부파일)를 분석시킬 때 어떤 위험이 있는가 3. 클로드가 악의적 지시를 따르지 않도록 어떻게 설계되어 있는가 4. 사용자 입장에서 프롬프트 인젝션을 예방하는 3가지 실천 방법 추가로: - 클로드 Computer Use 사용 시 브라우저에서 읽어온 악성 콘텐츠가 클로드의 행동에 영향 줄 수 있는지 - 이를 방어하기 위해 어떤 설정/프롬프트를 쓰면 좋은지

업무 유형별 안전 활용 가이드
나의 구체적 업무에 맞는 AI 보안 체크리스트
내 업무 유형에 맞는 AI 안전 활용 체크리스트를 만들어줘. 내 업무: [아래 중 해당하는 것 선택 또는 직접 입력] ━━ 선택지 예시 ━━ □ 법무/계약 업무 (계약서 검토, 법률 문서) □ 재무/회계 (재무제표, 예산 분석) □ HR/인사 (직원 정보, 채용, 급여) □ 영업/마케팅 (고객 정보, CRM 데이터) □ IT/개발 (소스코드, 인프라 접근 정보) □ 의료/헬스케어 (환자 정보, 임상 데이터) □ 교육 (학생 정보, 성적 데이터) 만들어줄 내용: 1. 이 업무에서 AI에게 절대 넘기면 안 되는 정보 (구체적인 예시 5가지 이상) 2. 이 업무에서 AI를 안전하게 활용할 수 있는 방법 (익명화 패턴, 대체 입력 방법 포함) 3. 이 업무 특성상 주의해야 할 법적 규정 (개인정보보호법, GDPR, HIPAA 등 해당하는 것) 4. 매일 AI 사용 전 30초 체크리스트 (3~5개 항목, 즉시 확인 가능한 것) 체크리스트 형식으로, 실무에서 바로 사용 가능하게.

보안 인시던트 대응 — 실수로 민감 정보 입력 시
실수로 민감 정보를 입력했을 때 즉시 해야 할 것
실수로 Claude에 민감한 정보를 입력했어. 지금 당장 해야 할 조치를 알려줘. 발생 상황: [아래 중 해당하는 것 입력] Case A: 고객 개인정보(이름, 이메일, 전화번호)를 포함한 데이터를 분석 요청에 그대로 붙여넣었어 Case B: API 키나 비밀번호가 포함된 코드를 입력했어 Case C: 기밀 계약서 원본 파일을 업로드했어 각 케이스에 대해 알려줄 것: 1. 즉시 해야 할 응급 조치 (순서대로) 2. 해당 대화를 삭제해도 실제로 완전히 사라지는 것인지 3. 관련 당사자(고객, 상사, 법무팀)에게 언제 보고해야 하는지 4. 같은 실수가 반복되지 않도록 하는 예방책 5. Anthropic에 삭제 요청을 해야 하는지와 방법 중요: 패닉하지 않도록 차분하고 실용적으로 알려줘. 현실적으로 가능한 것과 불가능한 것을 구분해서.

📌 27일차 핵심 정리

핵심 정책 변화 (2025년 9월)

Free/Pro/Max는 이제 기본으로 데이터 학습에 사용. 직접 꺼야 함. Team/Enterprise/API는 학습 미사용 보장.

지금 당장 해야 할 것

claude.ai → Settings → Privacy → "Help improve Claude" 토글 OFF. 그리고 시크릿 대화 위치 확인.

절대 입력 금지

주민번호·카드번호·비밀번호·API 키·미공개 재무 정보·환자 의료 기록 원본. 항상 익명화 후 사용.

익명화 3원칙

이름 → 역할명, 수치 → 비율, 코드의 민감정보 → 플레이스홀더. 시크릿 모드 + 익명화 = 이중 보호.

팀 관리

입력 금지 항목 명문화 + 승인 도구 목록 + AI 결과물 검토 의무화 + 분기별 정책 업데이트.
